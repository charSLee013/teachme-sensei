Model core / single-stream DiT

Transformer Architecture

Ideogram4Transformer 是一个 34 层单流 velocity predictor。它接收 Qwen3-VL 多层文本特征、noisy image latent tokens、timestep、3D position ids 和 segment/indicator masks，输出 image token velocity。

Abstract

本章不讲“Transformer 是什么”这种泛化定义，而是把这个仓库里的核心网络拆成可核对的结构：输入张量、34 层 block、Qwen3-VL 条件注入、MRoPE、AdaLN 和输出 velocity 约束。

Claim and Evidence

Claim

The implementation defines a 34-layer Transformer with 4608-dimensional embeddings, 18 attention heads, QK RMSNorm attention, MRoPE, SwiGLU MLP, timestep-driven AdaLN-style modulation, and a final velocity projection to 128 channels.

Evidence. `Ideogram4Config`, `Ideogram4Attention`, `Ideogram4TransformerBlock`, `Ideogram4Transformer.forward`.

Boundary. This page describes inference architecture and tensor contracts. It does not prove private training architecture variants beyond this released code.

Block Computation

image latent x -> input_proj -> image hidden Qwen features -> RMSNorm -> llm_cond_proj -> text hidden timestep -> sinusoidal embedding -> adaln_proj -> block modulation hidden = image hidden + text hidden + image_indicator_embedding for 34 blocks: QKV attention with QK RMSNorm + MRoPE + segment mask gated residual from AdaLN modulation SwiGLU MLP final layer -> 128-channel velocity

Qwen3-VL Conditioning

The pipeline extracts hidden states from 13 Qwen3-VL layers: 0, 3, 6, 9, 12, 15, 18, 21, 24, 27, 30, 33, 35. These states are stacked and reshaped into a concatenated feature vector before projection into the DiT hidden width.

Boundary

The source proves which layers are tapped and how features are concatenated. It does not provide an ablation proving the individual contribution of each tapped layer.

Output Contract

The final output has the same channel width as image latent tokens. The positive branch discards text positions after the forward pass; the negative branch is image-only. The sampler therefore updates only the image latent sequence.
