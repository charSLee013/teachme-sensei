Transformer 架构

Ideogram4Transformer 是一个 34 层单流速度预测器。它接收 Qwen3-VL 多层文本特征、带噪图像 latent token、时间步、三轴 position id 和 segment/indicator 掩码，输出图像 token 的速度更新量。

这个实现公开的是一个 34 层、单流的速度预测器。它把文本条件特征、图像 latent token、时间步和位置/角色信息放进同一条 Transformer 计算链路里，最后输出每个图像 token 的速度更新量。

这为什么重要。 只有把“它到底预测什么、条件从哪里进、残差和位置如何调制”这三件事看清，采样章节里的每一步更新才有落点。

为什么可以相信这一点。 参数表、张量形状表、模块计算图和条件注入说明都在描述同一套公开实现，而不是各说各话。

学习边界。 这一页讲的是已发布推理结构和张量契约，不把它外推成私有训练系统的全部架构细节。

参数表

字段数值作用
`emb_dim`4608Transformer 隐藏宽度。
`num_layers`34Transformer 模块数量。
`num_heads`18注意力头数；每个 head 的维度是 256。
`intermediate_size`12288SwiGLU MLP 的中间宽度。
`adanln_dim`512timestep 调制投影宽度。
`in_channels`128VAE 32 通道乘以 2 x 2 patch 后得到的输入宽度。
`llm_features_dim`4096 x 13Qwen3-VL 隐状态宽度乘以选中层位点数。

张量形状表

张量形状契约
`llm_features``(B, L, 4096*13)`文本特征槽位承载 Qwen 选中层状态；非文本槽位补零。
`x``(B, L, 128)`图像 latent token；正向分支里的文本槽位以零填充形式存在。
`t``(B,)` 或 `(B, L)`用于 AdaLN 风格调制的流匹配时间步。
`position_ids``(B, L, 3)`MRoPE 坐标。
`segment_ids``(B, L)`注意力分组掩码。
`indicator``(B, L)`区分 LLM token 与图像 token 的角色掩码。
`out``(B, L, 128)`速度输出；只有图像 token 位置在语义上会被继续使用。

模块计算流程

图像 latent x -> 输入投影 -> 图像隐藏状态 Qwen 条件特征 -> RMSNorm -> llm_cond_proj -> 文本隐藏状态 timestep -> 正弦时间嵌入 -> adaln_proj -> 模块调制向量 hidden = 图像隐藏状态 + 文本隐藏状态 + 图像角色嵌入 重复 34 个模块: QKV 注意力 + QK RMSNorm + MRoPE + 分段掩码 来自 AdaLN 调制的门控残差 SwiGLU 前馈层 最终层 -> 128 通道速度输出

Qwen3-VL 条件注入

pipeline 会从 13 个 Qwen3-VL 层位点抽取隐状态，再把它们拼成一条更宽的条件特征向量，最后投影到 DiT 的隐藏维度里。这说明文本条件不是只拿最后一层，而是利用多层信息共同构成条件表示。

请注意

公开实现能证明抽取了哪些层、如何拼接，但不能仅凭这一点断定每个层位点各自贡献了多少效果。

MRoPE 与 AdaLN

机制实现含义不变量
MRoPE根据三轴 `position_ids` 计算 rotary cos/sin，并把图像 H/W 频率交错塞进指定 section。`position_ids` 的最后一维必须是 3。
QK RMSNorm在注意力计算前对 query 和 key 的 head 向量做归一化。隐状态宽度必须能整除 head 数。
AdaLN modulation由时间步嵌入生成 attention 和 MLP 残差所需的 scale/gate。`adanln_dim` 必须与模块调制输入匹配。
SwiGLU`w2(silu(w1(x)) * w3(x))` 形式的前馈路径。中间宽度由配置和权重共同固定。

输出契约

最终输出与图像 latent token 保持相同的通道宽度。正向分支前向结束后会丢弃文本槽位，只把图像位置上的速度更新量交回采样器；负向分支则本来就是纯图像分支。也就是说，采样器真正更新的始终是图像 latent 序列。
