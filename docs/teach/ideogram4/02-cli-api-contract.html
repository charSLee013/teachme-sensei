CLI 接口契约

本页把 `run_inference.py` 解释为接口契约：每个参数都有默认值、运行影响、结果影响和失败模式。目标是让读者知道改变参数会改变哪一段推理系统。

课程主线 Main Line

本章主线很简单：CLI 不是一层薄包装，它决定设备、权重路径、prompt 扩展、安全检查和采样参数，因此也是读懂整条推理链路的最快入口。

现在记住 Remember Now

现在记住 Remember Now

CLI 是完整推理路径的用户入口，但它不只是转发 prompt。它决定设备、量化权重、prompt expansion、安全筛查、采样 preset、seed 和输出位置。

这为什么重要。 这些参数不是装饰项，改的是实际运行路径：是否先扩展 prompt、去哪个权重仓库、在哪里做安全检查，以及用哪条采样设定。

为什么可以相信这一点。 CLI 帮助文本、默认值和执行顺序相互吻合，说明这张参数表对应的是仓库真实暴露给用户的接口。

学习边界。 这一页只解释接口层行为；每一步内部如何实现，会在 token packing、sampling 和 architecture 章节展开。

参数契约表

参数默认值作用失败模式与联动
`--prompt`必填输入文本；开启 magic prompt 时会先扩展后再进入模型。缺值会让 argparse 直接报错。
`--output``output.png`保存第一张生成图的文件路径。文件系统写入错误通常发生在生成之后。
`--height`1024参与决定图像 token 网格和 schedule 分辨率。pipeline 要求能被 16 整除。
`--width`1024参与决定图像 token 网格，并影响 magic prompt 的宽高比信息。pipeline 要求能被 16 整除。
`--sampler-preset``V4_QUALITY_48`设置 `num_steps`、guidance schedule、`mu`、`std`。可选项受 `PRESETS` 键集合限制。
`--seed`0为 latent 噪声生成器设定可复现种子。代码、硬件或依赖变化后，相同 seed 也不保证同图。
`--device`CUDA，否则 MPS，再否则 CPU决定 pipeline 组件和采样张量使用的设备。nf4 路径会拒绝非 CUDA 设备。
`--quantization`CUDA: nf4；否则 fp8选择 Hugging Face 仓库和量化加载路径。需要受限权重访问权限，设备不兼容时会失败。
`--magic-prompt` / `--no-magic-prompt`开启控制是否把普通 prompt 扩展成结构化 caption。开启但没有 key 时，CLI 会在模型加载前退出。
`--magic-prompt-model``ideogram-4-v1`选择 Ideogram hosted API 或 OpenRouter Claude 实现。后端要求对应 API key 和可用网络。
`--magic-prompt-key``MAGIC_PROMPT_API_KEY` 或 `IDEOGRAM_API_KEY`用于 prompt 扩展服务认证。除非使用 `--no-magic-prompt`，否则必须提供。
`--warn-on-caption-issues`false把 caption verifier 失败从错误改成告警。可能让不符合 schema 的 prompt 继续进入模型。
`--hive-text-key``HIVE_TEXT_MODERATION_KEY`在生成前启用 prompt moderation。缺 key 时会 warning，并跳过这层筛查。
`--hive-visual-key``HIVE_VISUAL_MODERATION_KEY`在保存前启用图像 moderation。缺 key 时会 warning，并跳过输出筛查。

运行顺序

解析参数 -> 可选的 Hive 文本审核 -> 可选的 magic prompt 扩展 -> 选择采样预设 -> 从选定量化仓库加载 Ideogram4Pipeline -> pipe(prompt, height, width, preset params, seed) -> 可选的 Hive 图像审核 -> 保存第一张 PIL 图像

这条顺序本身就是接口契约：如果在加载 pipeline 之前失败，就不会进入权重下载和采样；如果在生成之后才失败，前面的推理资源已经消耗掉了。
