Token Packing 机制

Ideogram 4 的推理输入不是单一文本向量。Pipeline 构造一个 packed sequence：左侧是 Qwen3-VL 文本 token 对应的 hidden features，右侧是 image latent tokens。并行的 position、segment 和 indicator 轨道定义了每个 token 在 Transformer 中的角色。

概览

本章围绕 packed sequence 解释推理输入的真实结构：文本特征、图像 latent token、位置轨道、段轨道和 indicator 轨道如何共同决定 Transformer 看到的序列。

结论与依据

要记住的结论

`_build_inputs` 会构造一个 packed sequence：前半段是 text token slots，后半段是 image latent token slots。同时它还构造 Transformer 使用的 `position_ids`、`segment_ids` 和 `indicator` tensors。

依据。 `pipeline_ideogram4.py::_tokenize`, `_build_inputs`, `_encode_text`; constants `LLM_TOKEN_INDICATOR`, `OUTPUT_IMAGE_INDICATOR`, `IMAGE_POSITION_OFFSET`.

学习边界。 本页解释推理阶段的 tensor 构造，不推断训练 batch 的内部构造方式。

先定义对象

ObjectShape / valueRole
`token_ids``(B, total_seq_len)`Qwen text token ids; image slots stay zero.
`position_ids``(B, total_seq_len, 3)`MRoPE coordinates for text and image tokens.
`segment_ids``(B, total_seq_len)`Block-diagonal attention grouping and padding separation.
`indicator``(B, total_seq_len)`Distinguishes LLM tokens from output image tokens.
`num_image_tokens``height/16 * width/16`Image latent token count after VAE compression and 2x2 patching.

交互式 2D 序列通道

下面的通道图展示一个 prompt 如何变成并行 token metadata。拖动 text length 和 latent grid，可以看到 sequence length 与 image token budget 如何变化。

文字说明

序列顺序是 text tokens 在前，image tokens 在后。Text positions 是 `[pos, pos, pos]`，image positions 是 `[0, h, w] + 65536`。Indicator 轨道用 `LLM_TOKEN_INDICATOR` 标记文本位置，用 `OUTPUT_IMAGE_INDICATOR` 标记图像位置。

3D 网格理解

3D 视图是示意图：它把序列角色映射成空间隐喻。Text tokens 是抬高的轨道，image tokens 是 H x W 平面。旋转 lattice 可以区分 sequence role、grid geometry 和 token metadata。

3D 文字说明

抬高的蓝色轨道表示携带 Qwen-derived features 的文本位置；红色平面表示带 grid coordinates 的 image latent tokens。这只是解释用的示意投影，不是 literal embedding space 或 learned geometry。

机制后果

  • 提高分辨率会让 image token count 按平方级增长。
  • 更长的 prompt 会增加 text sequence length，但不会改变 image grid size。
  • Text/image role 不是只靠 position 推断；`indicator` 明确参与 masking 和 embeddings。
  • Padding 通过 segment ids 保持隔离，不会混入真实 sample sequence。
