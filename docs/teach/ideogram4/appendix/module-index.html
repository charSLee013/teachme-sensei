系统层地图

系统层主要模块职责契约课程跳转
用户入口 / API`run_inference.py`, `__init__.py`暴露 CLI 入口和公开导入接口。CLI 接口契约
提示协议`magic_prompt.py`, `caption_verifier.py`把 prompt 转成结构化条件，或检查 prompt 是否满足结构化约束。JSON Caption 协议
表示层`pipeline_ideogram4.py`, `constants.py`完成 tokenization、Qwen 特征提取、文本/图像 token 打包，以及位置与角色轨道分配。Token 打包
模型核心`modeling_ideogram4.py`依据文本特征、带噪 latent、timestep、位置和掩码预测图像 token 的速度更新量。Transformer 架构
采样层`scheduler.py`, `sampler_configs.py`, 采样主循环根据 logit-normal schedule 生成步进区间，并用 Euler 更新积分速度。采样与 CFG
解码层`autoencoder.py`, `latent_norm.py`把 latent token 反归一化并还原成 VAE latent 图，再解码成 RGB 图像。输出契约
运行时`quantized_loading.py`把 nf4/fp8 的量化权重映射成可执行的 PyTorch 模块。模型与运行时
安全与治理`safety.py`, `docs/safety.md`, licenses在有密钥时执行 prompt/output moderation，并说明使用边界与许可约束。安全、访问与许可

不变量与失败模式

系统层不变量失败模式
表示层height 和 width 必须能被 `patch_size * ae_scale_factor = 16` 整除。pipeline 会在进入采样前直接报错。
文本编码文本 token 数量不能超过 `max_text_tokens = 2048`。超过上限会在进入 Qwen 编码前报错。
nf4 运行时bnb 4-bit 路径要求 CUDA 设备。在非 CUDA 设备上构建 transformer 会失败。
Caption 校验器默认 pipeline 会把 verifier 问题当成错误处理。直接传入普通文本时，如果不用 magic prompt 或 warning mode，就可能失败。
Hive 安全层运行时过滤依赖可用密钥。缺少密钥时会发出 warning，但不会自动阻断。
