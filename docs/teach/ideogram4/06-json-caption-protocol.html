Prompt 协议 / 结构化条件

JSON Caption Protocol

Ideogram 4 的 prompt 设计重点是把画面条件结构化：整体描述、风格、媒介、颜色、背景、对象、文字和可选位置约束被组织成 JSON caption。magic prompt 在 CLI 默认路径中承担 plain text 到 structured caption 的编译器角色。

概览

本章解释的是这个仓库里真正可执行的 prompt 协议：JSON caption 的字段语义、约束关系，以及为什么 magic prompt 会把自然语言先编译成结构化条件。

结论与依据

要记住的结论

该仓库把结构化 JSON caption 作为高控制力 prompt protocol，并提供 verifier 检查 schema、key order、bounding boxes 和 color palette constraints。

依据。 `docs/prompting.md`, `caption_verifier.py`, `magic_prompt.py`, CLI magic-prompt default behavior.

学习边界。 与其他 prompting 范式的比较只用于理解设计取舍，不推断 Ideogram 私有训练过程或竞品内部实现。

Schema 语义

FieldCondition representedWhy it matters
`high_level_description`Global semantic intent.Gives the model a concise whole-image target.
`style_description.aesthetics`Visual mood and style adjectives.Separates style from object content.
`lighting`Illumination condition.Controls scene rendering independent of object list.
`photo` / `art_style`Photographic or non-photographic rendering mode.Enforces mutually exclusive style branch in verifier.
`color_palette`Global or per-element colors.Makes palette control explicit instead of implicit prose.
`background`Scene environment.Separates context from foreground elements.
`elements[]`Objects or text instances, optional bbox.Gives element-level composition and typography targets.

Prompt 范式对比

ParadigmControl formStrengthTradeoff
Plain promptNatural-language sentence.Low writing overhead.Layout, palette, and typography control remain implicit.
Prompt + negative promptPositive/negative prose.Can suppress unwanted concepts.Still weakly structured and often heuristic.
External control modulesPrompt plus extra image/control condition.Can provide strong spatial control.Control is distributed across tools and assets.
Ideogram JSON captionStructured fields for style, elements, text, palette, optional bbox.Centralizes composition and design controls in one text protocol.Higher authoring burden; verifier/magic prompt becomes important.

把 Magic Prompt 理解为编译器

在 CLI 默认路径中，plain prompt 不会直接送入模型，而是先通过注册的 magic prompt backend 展开成 minified JSON caption。可以把 magic prompt 理解为从自然语言意图到结构化模型条件的编译器。

失败条件

如果启用 magic prompt 但没有可用 API key，CLI 会在加载模型前退出。如果禁用 magic prompt，调用方必须提供 verifier 可接受的 prompt，或显式使用 warning mode。
