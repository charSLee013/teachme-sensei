Ideogram 4 推理系统课程

这门课程讲解 Ideogram 4 推理仓库的核心结构：模型接口、权重加载、采样逻辑、提示词处理、安全过滤和本地部署约束。课程以源码、项目文档和可访问官方资料为依据，帮助读者理解哪些结论来自代码，哪些来自公开资料，哪些只是用于理解机制的教学重建。

课程主线

要记住的结论

读完后，读者应能说明 Ideogram 4 本地推理从 CLI 参数进入 pipeline，再经过 Qwen3-VL 条件编码、单流 Transformer velocity prediction、Euler flow-matching sampling、VAE decode 和可选安全过滤的完整路径。

依据。 课程主要依据 `run_inference.py`、`src/ideogram4/*`、`docs/*` 和 README；权重访问方式以项目说明和 Hugging Face gated repository 行为为边界。

学习边界。 课程会区分源码事实、公开声明、机制解释和未公开信息。训练数据、优化器、训练时长、训练算力和 post-training 细节没有在当前公开材料中给出，因此不会被当作事实讲解。

依据分层

层级课程中的用途读者应如何理解
源码事实直接来自本地源码或文档的行为、参数、结构。可以作为当前 checkout 的事实。
公开声明发布方 README、官方博客、模型卡中的公开说明。可以引用为公开资料，不等同于独立复现。
背景知识与 prompt、CFG、flow matching、model card 等通用范式的对比。用于定位设计，不证明私有训练选择。
教学重建为了理解推理机制而整理出的解释路径。帮助理解机制，不能当作官方训练配方。
未公开信息当前公开资料和本地源码没有确认的信息。不把缺失信息写成事实。

学习顺序

1. Model / Runtime Card

先确定模型身份、变体、输入输出、硬件、许可、安全和限制。

2. CLI / API Contract

解释每个 `run_inference.py` 参数如何影响运行和结果。

3. Token Packing

用 2D/3D 图解释 prompt、text token 和 image latent token 如何进入同一序列。

4. Sampling / CFG

定义变量、公式和每步更新，说明条件信号如何改变采样方向。

5. Transformer Architecture

深入 shapes、Qwen taps、MRoPE、AdaLN、SwiGLU 和 output contract。

6. JSON Caption Protocol

解释结构化 prompt 的语义、约束和与其他 prompt 范式的差异。

7. 公开训练信息

说明公开资料能支持哪些结论，以及哪些训练细节仍未公开。

学习边界

学习范围

这门课聚焦源码和公开资料能够支持的推理系统知识。涉及训练数据、优化器、训练算力、训练时长或 benchmark 复现时，课程只说明公开资料已经给出的内容；没有公开依据的部分会明确标为未公开信息。
