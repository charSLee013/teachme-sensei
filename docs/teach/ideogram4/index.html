Ideogram 4 推理系统课程

这套课程把仓库讲成一个可验证的推理系统：先看模型与运行时契约，再看 token 序列、采样、Transformer、prompt 协议和公开训练信息。读者读完后应当能够复述主线，知道依据来自哪里，也知道哪些地方目前还不能确认。

课程主线 Main Line

现在记住 Remember Now

该仓库公开的是 Ideogram 4 的推理侧系统：CLI 入口、推理 pipeline 编排、Qwen3-VL 条件编码、单流 Transformer 速度预测器、Euler 流匹配采样器、VAE 解码、量化权重加载、提示协议和可选安全过滤。

这为什么重要。 先把它当成推理工具包来读，后面各章的模型卡、CLI、token 序列、采样和架构才会落到同一条主线上。

为什么可以相信这一点。 入口脚本、核心模块和仓库文档相互印证：它们都围绕本地推理、量化权重加载、条件编码、采样和安全集成展开。

学习边界。 本课程聚焦已经公开的推理接口和公开训练说明。训练数据配方、优化器、训练算力和训练时长目前没有公开资料可确认，因此不会被当作既定事实讲解。

本课程补上的内容

学习结构

每章先给主线，再解释依据、机制和学习边界，让技术读者先抓住系统结构，再理解证据和限制。

接口与运行时

CLI 参数、权重变体、许可、安全和硬件约束被整理成可操作的契约，而不是散落在 README 和代码里。

机制解释

Token 打包和采样/CFG 用交互图、变量表和文字备选说明来解释，不靠术语堆砌。

训练边界

课程把已公开说明、源码能确认的推理行为和当前仍待公开的部分分开讲清，避免把空白处讲成事实。

如何理解依据

依据层级课程中的用途读者应如何理解
源码和文档直接说明的内容来自可见实现、接口定义和仓库文档的行为、参数和结构。可以把它们当作这个 checkout 的已知事实。
发布方公开说明README、官方博客和公开模型说明里的发布信息。它们说明发布口径，不等同于我们已经独立复现。
相关范式对照与 prompt、CFG、flow matching、model card 等通用做法的比较。用于帮助定位设计，不代表已经证明 Ideogram 的私有训练选择。
为理解机制而做的重述把推理过程拆成更容易学会的解释步骤。用于教学，不把它当作官方训练配方。
当前公开资料还不能确认的部分本地源码和公开资料都没有回答的问题。保持边界，不用猜测来补空白。

阅读顺序

1. 模型与运行时

先确定模型身份、变体、输入输出、硬件、许可、安全和限制。

2. CLI 接口契约

解释每个 `run_inference.py` 参数如何影响运行和结果。

3. Token 打包

用 2D/3D 图解释提示文本、文本 token 和图像 latent token 如何进入同一序列。

4. 采样与 CFG

定义变量、公式和每步更新，说明条件信号如何改变采样方向。

5. Transformer 架构

深入张量形状、Qwen 取层位点、MRoPE、AdaLN、SwiGLU 和输出契约。

6. JSON Caption 协议

解释结构化提示协议的语义、约束和与其他提示范式的差异。

7. 公开训练信息

把已公开的训练说明、源码能确认的推理事实和当前仍待公开的部分分开讲清。

学习边界

请注意

本课程把重点放在已公开的模型接口、运行时约束、条件生成机制和可确认的训练公开说明上。真实图片质量、官方基准测试复现，以及未公开训练配方都需要额外权重、实验或一手资料，当前页面不会把这些内容写成已验证结论。
