Ideogram 4 推理系统技术报告

本报告将该仓库界定为 Ideogram 4 的推理工具包：它集成模型结构、权重加载、采样逻辑、提示词处理、安全过滤和说明文档，面向本地推理与部署集成场景。本报告基于本地源码、项目文档和可访问官方资料，对其模型接口、运行时约束、条件生成机制和公开训练信息进行分层说明。

Abstract

Claim

该仓库公开的是 Ideogram 4 的推理侧系统：CLI 入口、pipeline 编排、Qwen3-VL 条件编码、单流 Transformer velocity predictor、Euler flow-matching sampler、VAE decode、量化权重加载、prompt protocol 和可选安全过滤。

Evidence. 本地 `run_inference.py`、`src/ideogram4/*`、`docs/*` 和 README。权重访问边界由 README 和未认证 Hugging Face raw fetch 的 401 响应共同支持。

Boundary. 训练数据、optimizer、训练时长、训练 compute、post-training 细节不由本仓库源码公开；本报告仅按公开资料、源码事实、教学重建和 unknown 层处理训练内容。

Reading Order

1. Model / Runtime Card

先确定模型身份、变体、输入输出、硬件、许可、安全和限制。

2. CLI / API Contract

解释每个 `run_inference.py` 参数如何影响运行和结果。

3. Token Packing

用 2D/3D 图解释 prompt、text token 和 image latent token 如何进入同一序列。

4. Sampling / CFG

定义变量、公式和每步更新，说明条件信号如何改变采样方向。

5. Transformer Architecture

深入 shapes、Qwen taps、MRoPE、AdaLN、SwiGLU 和 output contract。

6. JSON Caption Protocol

解释结构化 prompt 的语义、约束和与其他 prompt 范式的差异。

7. Training Public Record

正向分层公开训练声明、源码支持的推理事实、教学重建和 unknown。

Limitations

Boundary
