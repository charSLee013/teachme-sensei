模型接口 / 运行时约束

模型与运行时

本页将 Ideogram 4 推理包描述为可运行模型接口，而不是概念图。字段覆盖模型身份、变体、输入输出、运行时、许可、访问、安全与限制。

概览

要记住的结论

该仓库提供 Ideogram 4 的本地推理接口：输入为 caption string 和图像尺寸/采样参数，输出为 `PIL.Image`。模型权重通过 Hugging Face gated repositories 获取，CLI 暴露 nf4 和 fp8 两类变体。

依据。 `run_inference.py` 的 `QUANTIZATION_REPOS`、`Ideogram4PipelineConfig`、README model zoo 和 license files。

学习边界。 未认证环境无法读取 Ideogram HF raw model cards；本页只使用本地源码、README 和可访问官方页面中的信息。

模型身份

名称Ideogram 4
仓库包`ideogram-4` Python 推理包
公开参数量本地 README model zoo 中标注为 9.3B
主要任务从 prompt/caption string 生成图像
实现范围推理 pipeline、模型架构、量化加载、prompt 处理、安全 API 调用和文档

变体与运行时

变体权重仓库加载路径运行时约束
nf4`ideogram-ai/ideogram-4-nf4`bitsandbytes `Linear4bit` 和预量化 state dict当前实现要求 CUDA。
fp8`ideogram-ai/ideogram-4-fp8`weight-only e4m3 FP8 buffers + per-row scaleActivation 使用 compute dtype；代码路径不只服务 FP8 tensor cores。

输入与输出

字段契约影响
PromptString；推荐使用结构化 JSON caption。经 Qwen3-VL hidden-state extraction 后提供文本条件。
Height / width实现要求能被 16 整除。决定 image token grid 和 schedule resolution 调整。
Sampler parametersSteps、guidance schedule、`mu`、`std`。控制 denoising trajectory 和条件强度。
Output`PIL.Image` 对象列表。CLI 会把第一张图保存到 `--output`。

安全、访问与许可

访问

README 说明模型权重位于 Hugging Face gated repositories；未配置凭据时无法直接读取 raw HF README。

许可

源码许可证是 Apache-2.0；模型权重使用 `model_licenses/` 下的 Ideogram Non-Commercial Model Agreement。

安全

Hive moderation 已接入，但依赖 API key；缺少 key 时 CLI 给出 warning，而不是完成阻断式审核。

已知边界

  • 训练数据、optimizer、compute 和训练时长没有在该推理仓库中公开。
  • 真实图像质量、延迟和显存表现需要有权限的权重与具体硬件测量。
  • Prompt safety 和 output safety 依赖已配置的 Hive keys 或等价部署控制。
