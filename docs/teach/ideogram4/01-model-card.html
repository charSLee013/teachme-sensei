模型与运行时

本页将 Ideogram 4 推理包描述为可运行模型接口，而不是概念图。字段覆盖模型身份、变体、输入输出、运行时、许可、访问、安全与限制。

课程主线 Main Line

现在记住 Remember Now

这份仓库交付的是 Ideogram 4 的本地推理接口：输入是 prompt 与尺寸/采样参数，输出是生成后的图像对象；公开提供 nf4 和 fp8 两类运行时路径。

这为什么重要。 先搞清模型身份、变体、许可和运行时，后面再读 CLI、采样和架构章节时，才不会把系统约束误解成实现细节。

为什么可以相信这一点。 公开 README、下载说明、运行时配置和许可文件在同一方向上描述了这些接口与变体。

学习边界。 这里讲的是已发布推理包和可见许可边界，不把受限权重仓库之外的私有信息补写成事实。

模型身份

名称Ideogram 4
仓库包名`ideogram-4` Python 推理包
公开参数规模本地 README model zoo 标注为 9.3B
主要任务依据 prompt/caption string 进行文生图生成
实现范围推理 pipeline、模型结构、量化加载、prompt 处理、安全接口和配套文档

变体与运行时

变体权重仓库加载路径运行时约束
nf4`ideogram-ai/ideogram-4-nf4`bitsandbytes `Linear4bit` 与预量化 state dict当前实现要求 CUDA。
fp8`ideogram-ai/ideogram-4-fp8`weight-only e4m3 FP8 buffer 与逐行 scale激活值沿用 compute dtype；这条路径不等于只支持 FP8 tensor core。

输入与输出

字段契约作用
提示文本字符串；更推荐结构化 JSON caption。在 Qwen3-VL 隐状态提取之后提供文本条件。
高度 / 宽度当前实现要求能被 16 整除。决定图像 token 网格和 schedule 的分辨率调整。
采样参数步数、guidance schedule、`mu`、`std`。控制去噪轨迹和条件强度。
输出结果`PIL.Image` 对象列表。CLI 会把第一张图保存到 `--output`。

安全、访问与许可

访问

README 说明权重通过 Hugging Face 的受限访问流程发放，因此阅读公开源码和直接获取完整权重元信息是两件事。

许可

源码使用 Apache-2.0；模型权重采用 Ideogram Non-Commercial Model Agreement，对商业使用边界有单独限制。

安全

Hive moderation 已接入，但要依赖可用的密钥配置。没有密钥时，CLI 会给出提示，而不会自动补出同等强度的过滤。

当前边界

  • 训练数据、优化器、训练算力和训练时长并不是这个推理仓库会公开回答的问题。
  • 真实图片质量、延迟和显存表现需要在已获授权的权重和具体硬件上做实测。
  • 提示词安全和输出安全仍然依赖 Hive 密钥或等效的部署侧控制。
