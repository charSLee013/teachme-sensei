Sampler layer / Euler flow matching

Sampling and Classifier-Free Guidance

采样循环从随机 latent 开始。每一步同时计算文本条件 velocity 和 image-only baseline velocity，然后用 guidance weight 放大条件方向，再用 Euler update 移动 latent 状态。

Abstract

本章把采样写成可追踪的更新规则：先看条件 velocity 和无条件 baseline 如何合成，再看 guidance weight、schedule 和 Euler 更新怎样共同改变 latent 轨迹。

Claim and Evidence

Claim

The implementation performs an Euler-style update over scheduled timesteps and combines conditional and unconditional velocity predictions at each step.

Evidence. `Ideogram4Pipeline.__call__`, `LogitNormalSchedule`, `make_step_intervals`, `PRESETS`.

Boundary. The visualization is schematic. It explains update direction and conditioning, not actual high-dimensional latent geometry.

Step Equation

Mechanism

The code writes guidance as `gw * v_pos + (1 - gw) * v_neg`. For explanation, the equivalent form is more direct:

v_guided = v_neg + gw_i * (v_pos - v_neg)
z_{i-1}  = z_i + v_guided * (s_i - t_i)

`v_pos - v_neg` is the condition direction. `gw_i` scales how strongly the text-conditioned direction modifies the image-only baseline.

Interactive Guidance Vector

The slider changes guidance weight. The red arrow is the guided update direction; larger guidance amplifies the condition component relative to the image-only baseline.

Pseudocode After Definitions

for i in reversed(range(num_steps)):
    t_i = schedule(step_intervals[i + 1])
    s_i = schedule(step_intervals[i])
    v_pos = conditional_transformer(text_features, z_i, t_i)
    v_neg = unconditional_transformer(zeros, z_i, t_i)
    v_guided = v_neg + gw_i * (v_pos - v_neg)
    z_{i-1} = z_i + v_guided * (s_i - t_i)

Consequences

  • Step count changes how many Euler updates are applied.
  • Guidance schedule changes the balance between prompt adherence and baseline image dynamics over time.
  • Resolution affects the logit-normal schedule mean through pixel count.
  • Actual quality tradeoffs require generation experiments; this page explains the mechanism, not benchmark outcomes.
