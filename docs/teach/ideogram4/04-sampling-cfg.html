采样层 / Euler 流匹配

采样与 CFG

采样循环从随机 latent 开始。每一步同时计算文本条件速度和纯图像基线速度，然后用引导权重放大条件方向，再用 Euler 更新推动 latent 状态前进。

课程主线 Main Line

本章主线是：采样不是神秘黑箱，而是一条步进更新规则。每一步都先比较文本条件和图像基线的速度，再按引导权重把它们合成到下一次 Euler 更新里。

采样循环会沿着一组 schedule 时刻前进，并在每一步同时得到文本条件速度和图像基线速度。真正用来更新 latent 的方向，是这两者按引导权重合成后的结果。

这为什么重要。 只要抓住“基线方向 + 条件修正”这条主线，就能理解 CFG 为何会改变 prompt 约束强度，也能看懂后面的公式和伪代码。

为什么可以相信这一点。 变量表、公式、伪代码和交互向量图都在解释同一条更新规则，只是视角不同。

学习边界。 页面里的向量图是示意图，用来解释更新方向和条件修正，不把高维 latent 空间误画成二维真几何。

符号含义实现对象

单步公式

机制

实现里把引导写成 `gw * v_pos + (1 - gw) * v_neg`。为了让作用方式更直观，下面改写成等价但更好读的形式：

v_guided = v_neg + gw_i * (v_pos - v_neg)
z_{i-1}  = z_i + v_guided * (s_i - t_i)

`v_pos - v_neg` 表示文本条件带来的修正方向；`gw_i` 决定这股修正要多强地偏转图像基线。

引导向量示意

拖动滑块会改变引导权重。红色箭头表示最终更新方向；引导越大，文本条件相对于图像基线的影响就越强。

文字说明

如果把蓝色箭头看成图像基线，把绿色箭头看成带文本条件的版本，那么红色箭头就是“先从蓝色出发，再向绿色方向偏过去”的结果。引导越大，这个偏转越明显。

伪代码

for i in reversed(range(num_steps)):
    t_i = schedule(step_intervals[i + 1])
    s_i = schedule(step_intervals[i])
    v_pos = conditional_transformer(text_features, z_i, t_i)
    v_neg = unconditional_transformer(zeros, z_i, t_i)
    v_guided = v_neg + gw_i * (v_pos - v_neg)
    z_{i-1} = z_i + v_guided * (s_i - t_i)

机制结果

  • 步数越多，Euler 更新次数就越多。
  • 引导日程会改变 prompt 约束和图像基线动态之间的平衡。
  • 分辨率会通过像素数量影响 logit-normal 日程的位置。
  • 真实画质取舍要靠生成实验验证；这一页解释的是机制，不是假定基准测试结果。
