采样层 / Euler flow matching

采样与 Classifier-Free Guidance

采样循环从随机 latent 开始。每一步同时计算文本条件 velocity 和 image-only baseline velocity，然后用 guidance weight 放大条件方向，再用 Euler update 移动 latent 状态。

概览

本章把采样写成可追踪的更新规则：先看条件 velocity 和无条件 baseline 如何合成，再看 guidance weight、schedule 和 Euler 更新怎样共同改变 latent 轨迹。

结论与依据

要记住的结论

实现会在 scheduled timesteps 上执行 Euler-style update，并在每一步合成 conditional 与 unconditional velocity predictions。

依据。 `Ideogram4Pipeline.__call__`, `LogitNormalSchedule`, `make_step_intervals`, `PRESETS`.

学习边界。 可视化是机制示意图，用来解释更新方向和条件信号，不代表真实高维 latent geometry。

先定义变量

SymbolMeaningImplementation object
`z_i`Current image latent tokens at step i.`z`
`t_i`Current schedule time.`t_val`
`s_i`Next schedule time.`s_val`
`v_pos`Velocity predicted with text conditioning.`pos_v`
`v_neg`Image-only baseline velocity.`neg_v`
`gw_i`Guidance weight for this step.`gw_per_step[i]`
`delta_i`Step interval.`s_val - t_val`

单步更新公式

机制

代码把 guidance 写成 `gw * v_pos + (1 - gw) * v_neg`。用于教学时，下面这个等价形式更直接：

v_guided = v_neg + gw_i * (v_pos - v_neg)
z_{i-1}  = z_i + v_guided * (s_i - t_i)

`v_pos - v_neg` 是 condition direction。`gw_i` 决定 text-conditioned direction 对 image-only baseline 的修正强度。

交互式 Guidance 向量

滑块改变 guidance weight。红色箭头是 guided update direction；guidance 越大，条件分量相对 image-only baseline 的放大越强。

定义之后再看伪代码

for i in reversed(range(num_steps)):
    t_i = schedule(step_intervals[i + 1])
    s_i = schedule(step_intervals[i])
    v_pos = conditional_transformer(text_features, z_i, t_i)
    v_neg = unconditional_transformer(zeros, z_i, t_i)
    v_guided = v_neg + gw_i * (v_pos - v_neg)
    z_{i-1} = z_i + v_guided * (s_i - t_i)

机制后果

  • Step count 决定会应用多少次 Euler updates。
  • Guidance schedule 会改变 prompt adherence 与 baseline image dynamics 的时间分布。
  • Resolution 会通过 pixel count 影响 logit-normal schedule mean。
  • 真实质量取舍需要 generation experiments；本页解释机制，不声称 benchmark 结果。
