Anima 方法论参考

主题

这个参考页服务于三个问题：

  • 2B operating point 如何从 scaling law 与高分辨率 token 成本来解释。
  • anime + non-anime art mixture 如何写成可辩护的数据分布问题。
  • warmup、beta2、text-side policy 如何形成可判别的实验矩阵。

公开事实

  • 公开 Diffusers 配置可确认：Cosmos transformer、Qwen3 text encoder、Anima text conditioner、Qwen-Image VAE、FlowMatch scheduler。
  • 训练数据公开边界：several million anime images + about 800k non-anime artistic images，no synthetic data。
  • 原始 optimizer、真实训练步数、完整过滤阈值未公开。

参数量选型

较稳妥的理论解释是：2B 更像一个 operating point，而不是孤立标签。它同时受以下量约束：

  • hidden width
  • transformer depth
  • latent token count
  • 分辨率预算
  • 文本条件维度

在高分辨率 latent self-attention 为二次复杂度的前提下，更大模型并不天然优于一个受到预算约束的 2B 级方案。

数据混合

公开材料表明目标分布不是纯 anime-only。一个可教学的写法是：

p_target = lambda_anime * p_anime + lambda_art * p_art
lambda_photo ~= 0

这里更重要的不是精确比例，而是三种竞争解释：

  • forgetting regularizer
  • caption bridge
  • aesthetic smoother

文本侧策略

text-side policy 至少包含三种竞争假设：

  • fully frozen
  • low-LR trainable
  • delayed unfreeze

它们真正区分开的，不是口头解释，而是：

  • 英文 prompt following
  • 中文 prompt following
  • attribute binding
  • rare concept retention
  • semantic drift

warmup 与稳定性

在 AdamW-like 解释框架中，warmup 更适合被理解为：

  • 帮助目标学习率进入稳定 regime
  • 与 beta2 的统计时间尺度耦合
  • 缓冲 early-loss volatility

把 warmup 仅仅写成“防止早期梯度爆炸”，对这类系统是不够的。
