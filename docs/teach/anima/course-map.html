Anima 课程地图

主线一句话

Anima 可以被理解为一个建立在公开 Cosmos 基座之上的 2B 级动漫向 text-to-image 系统；理解它，关键不在“背配置”，而在把架构、数据、文本条件与评估四条线一起看。

学习顺序

第 1 站：对象识别

读 lessons/01-what-is-anima.md

目标：

  • 知道 Anima 是什么，不是什么。
  • 知道哪些事实是公开的，哪些不是。

第 2 站：架构关系

读 lessons/02-architecture.md

目标：

  • 理解公开组件之间的关系。
  • 理解文本侧条件模块为什么会影响后续训练与提示词解释。

第 3 站：训练边界

读 lessons/03-training-boundaries.md

目标：

  • 理解 2B operating point、zero synthetic、mixture data、warmup 与稳定性之间的联系。

第 4 站：提示词机制

读 lessons/04-prompting.md

目标：

  • 理解哪些 prompting 规则来自公开 model card。
  • 区分“可确认的使用建议”和“尚待验证的机制解释”。

第 5 站：验收与实验

读 lessons/05-evaluation.md

目标：

  • 理解动漫模型为什么需要 FID 之外的指标。
  • 知道如何组织主结果、消融与 tail-sensitive evaluation。

深入路径

当你主线走完之后，再进入：

  • reference/methodology.md：参数量、数据混合、warmup 耦合的理论框架
  • reference/formulas.md：核心公式与符号表
  • reference/resources.md：论文、模型、工具入口
