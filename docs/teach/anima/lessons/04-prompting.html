Lesson 4：提示词、条件路径与公开可确认规则

本节记住什么

  • 当前关于 prompting 的主线，应围绕公开 model card 与公开配置来讲。
  • @artist、tag order、CFG 4-5、30-50 steps、512-1536 分辨率，属于公开可确认的使用建议。
  • 提示词页面必须区分公开建议与待验证机制。

为什么重要

提示词页面最容易把“经验技巧”“猜测机制”“公开事实”混成一团。这里如果不收紧边界，读者会以为已经知道了底层实现，但其实只是继承了旧假设。

课程位置

这是从“训练视角”转向“使用与条件机制视角”的一课。

用通俗语言讲清主线

公开材料支持我们稳妥地说出三类事情。

1. 公开可确认的使用建议

  • 推荐分辨率区间：512-1536
  • 推荐采样步数：30-50
  • 推荐 CFG：4-5
  • 训练提示词形态同时覆盖 Danbooru-style tags、natural-language captions 以及两者混合
  • 艺术家标签使用 @ 前缀时效果更稳定
  • 模型卡给出了一种 tag order：quality/meta/year/safety -> subject count -> character -> series -> artist -> general tags

2. 公开可确认的条件路径边界

公开配置中可确认的主 text encoder 是 Qwen3Model，并存在独立 AnimaTextConditioner。因此，提示词机制的主线解释应围绕当前公开组件来建立。

3. 仍然只属于解释性重构的部分

  • 为什么 @artist 前缀更有效
  • 为什么 tag 顺序会影响视觉权重分布
  • 为什么某些 prompt 片段更容易被前部 token 抢占注意力

这些都可能是合理机制解释，但在没有 tokenization 实验和 ablation 之前，更适合写成待验证解释，而不是既成实现事实。

已公开确认

  • 主 text encoder 是 Qwen3Model。
  • model card 给出 @artist、tag order、CFG 4-5 等使用建议。
  • 随机 tag dropout 存在，但精确比例未公开。

解释性重构

  • @ 前缀可能改变 tokenizer 行为或标签可识别性。
  • tag order 可能通过条件权重分配影响结果。

尚未公开确认

  • 公开 modular pipeline 中同时出现的 T5Tokenizer 在推理路径里承担的精确角色。
  • 某些旧的 token 级经验解释是否仍能局部迁移到当前系统。

自测

  1. 为什么 Qwen3Model 这一事实会改变 prompt 机制的主线解释？
  2. 哪些 prompting 规则属于公开建议，哪些只是机制推测？
  3. 为什么 @artist 的解释应该被写成“待验证”，而不是“底层已经证明”？
