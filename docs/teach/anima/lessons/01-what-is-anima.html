Lesson 1：Anima 到底是什么

本节记住什么

  • Anima 是一个 2B 级 text-to-image 模型，面向 anime、characters、styles 和 non-photorealistic art。
  • 它基于公开的 Cosmos 路线，而不是一个完全私有、不可见的黑盒。
  • 公开材料能确认一部分架构和数据边界，但仍有不少训练细节没有公开。

为什么重要

如果一开始就把 Anima 当成“又一个动漫 checkpoint”，后面会很容易把架构、数据和评估全部混在一起。先把对象边界讲清，后面的推导才不会漂。

课程位置

这是课程入口。目标不是讲细节，而是建立一张正确的地图。

用通俗语言讲清主线

可以把 Anima 先理解成三个叠在一起的层：

  1. 一个公开可见的生成基座。
  2. 一个被拉向动漫/插画分布的领域适配过程。
  3. 一组还没有完全公开的训练决策。

公开材料目前能直接确认的事实包括：

  • 它是 2B 参数量级。
  • 它的目标分布主要是 anime / characters / styles / non-photorealistic art。
  • 训练数据包含数百万 anime 图像，以及约 800k 非动漫艺术图像。
  • 训练没有使用 synthetic data。
  • Diffusers 配置公开了 Cosmos transformer、Qwen3 text encoder、Anima text conditioner、Qwen-Image VAE 和 FlowMatch scheduler。

这些事实已经足够建立一张可靠的入门地图，但许多训练细节仍然没有公开。

已公开确认

  • 主 text encoder 是 Qwen3Model。
  • 存在独立的 AnimaTextConditioner。
  • 存在 Qwen-Image VAE。
  • 生成建议区间包括 512-1536 分辨率、30-50 步、CFG 4-5。

解释性重构

  • 2B 可能是一个在高分辨率注意力成本下较现实的 operating point。
  • non-anime art 子集可能承担 regularization 或 caption bridge 作用。

尚未公开确认

  • 原始训练是否存在明确 Stage 1 / Stage 2。
  • text conditioner 是否在原始训练中始终冻结。
  • random tag dropout 的精确比例。

自测

  1. 为什么 “公开能确认一部分” 和 “很多训练细节仍未公开” 不是一回事？
  2. 为什么 zero synthetic data 对理解 Anima 的训练边界特别重要？
  3. 目前公开材料中，哪一个事实最能改变你对文本条件路径的理解？
