Lesson 3：训练边界与方法论主线

本节记住什么

  • Anima 的核心训练问题不是“怎么把 loss 跑下去”，而是怎么同时守住计算预算、文本语义、数据分布和收敛稳定性。
  • zero synthetic data、anime + non-anime art mixture、text-side policy、warmup 耦合，是这条主线上的四个关键杠杆。
  • 许多细节仍未公开，因此课程里必须把“已确认事实”和“解释性重构”分开。

为什么重要

论文真正有价值的部分，通常不在 config 表本身，而在你能否解释“为什么这个 operating point 合理、它牺牲了什么、又保住了什么”。

课程位置

这是从“组件识别”进入“训练推理”的过渡课。

用通俗语言讲清主线

可以把训练问题压缩成四组张力：

1. 参数量与分辨率张力

2B 不是抽象数字。它和 1024-1536px 区间下的 latent token 成本绑定在一起。分辨率越高，attention 成本上涨越快，因此更大的模型未必真的更优。

2. 数据混合与遗忘张力

公开材料显示，训练数据不是纯 anime-only，还混入了 non-anime artistic images。这让训练目标不只是“风格专业化”，还带上了“不要把较宽的视觉构图和语言对齐能力完全忘掉”的问题。

3. 文本侧适配与语义漂移张力

一旦 text-side 模块可训练，域内标签适配可能变强，但 prompt drift 风险也会同时上升。于是 fully frozen / low-LR trainable / delayed unfreeze 就变成了真正需要实验判别的假设空间。

4. warmup 与稳定性张力

warmup 不是一个“防爆炸按钮”，更像是为了让目标学习率与 Adam 二阶统计量进入更稳定关系的缓冲区。它和 beta2、分辨率 bucket、梯度波动一起决定 early-loss volatility。

已公开确认

  • zero synthetic data 是模型卡明确声明的事实。
  • 数据中存在 non-anime art 子集。
  • 模型卡给出 llm_adapter_lr=0 与 rank-32 2e-5 这类下游微调建议。

解释性重构

  • non-anime art 子集可能充当 forgetting regularizer 或 caption bridge。
  • 2B 级 DiT 很可能是高分辨率预算下的现实 operating point。
  • warmup 长度与 beta2 的关系，比“经验步数”更值得解释。

尚未公开确认

  • 原始 optimizer 与真实 warmup 步数。
  • 原始训练是否真的分阶段。
  • 原始数据混合采样器是 constant、annealed 还是 rarity-aware。

自测

  1. 为什么 zero synthetic data 不只是“品味问题”，而是分布稳定性问题？
  2. 为什么 non-anime art 子集既可能帮忙，也可能稀释 anime specialization？
  3. 为什么 text-side policy 必须用消融实验来判别，而不能仅靠直觉？
