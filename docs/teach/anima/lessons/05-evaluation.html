Lesson 5：评估、验收与消融

本节记住什么

  • 动漫模型评估不能只看 FID。
  • 公开基线之间的评估协议并不统一，因此“比较谁更强”之前要先比较“用了什么指标”。
  • 角色一致性、tail coverage、prompt following 和 human preference，都是动漫场景里不可省略的维度。

为什么重要

如果只盯着一个分数，你很容易把“看起来更接近训练分布”和“真正更懂动漫 prompt”混为一谈。

课程位置

这是主线的最后一课。它把架构、训练和数据问题收束到“如何验收”。

用通俗语言讲清主线

1. FID 只回答一部分问题

FID 衡量的是生成分布与参考分布的距离，但它默认使用的 Inception 特征来自写实图像语境，因此在动漫域会遇到 domain shift。它能提供整体分布信息，却不一定能看见 tail collapse、角色一致性或复杂 prompt 绑定失败。

2. prompt following 需要独立观察

CLIP 类指标可以帮助观察通用文本对齐，但对具体角色名、艺术家风格和罕见动漫概念并不总是足够敏感。因此动漫场景需要把“通用语义对齐”和“领域特异性控制”拆开看。

3. 角色一致性与 tail coverage 是动漫域特有痛点

角色一致性更接近“这个角色还是不是同一个角色”，而 tail coverage 更接近“低频角色名、稀有风格和小众构图有没有一起掉队”。这两类问题，通常不会被单独一个通用指标完整捕获。

4. human preference 不是可有可无

动漫图像的风格纯度、解剖规范、色彩协调、细节完成度，经常需要人工评价才能稳定识别。自动指标可以做筛查，但不能完全替代人工判断。

已公开确认

  • Illustrious-XL 与 NovelAI V3 的公开报告并没有共享统一的评估协议。
  • CCIP 代表了一类面向角色一致性的评估方向。

解释性重构

  • 一个更合适的动漫评估矩阵，应同时包含分布、对齐、角色、尾部和人工偏好。
  • tail-sensitive metric 在 zero synthetic / anti-collapse 研究里尤其重要。

尚未公开确认

  • Anima 官方完整 benchmark 套件。
  • 官方是否使用了动漫域替代版 FID。
  • 任何尚未跑出的 TBD 表格结果。

自测

  1. 为什么 FID 低不等于动漫生成质量一定高？
  2. 为什么 CCIP 这类指标在动漫域有独立价值？
  3. 为什么 TBD 或逆向估算值不应直接放进学生主线结论页？
