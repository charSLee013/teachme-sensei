FLUX.2 Teaching Pack

FLUX.2 教学包总地图

这不是“读完几个 HTML 就算结束”的速读包，而是一套从仓库真相出发、最终过渡到 VAE / 训练研究准备的课程系统。 你可以把它当作一条中文学习主线：先看清仓库是什么，再弄懂契约和对象，最后决定研究起点与迁移方向。

先记住： 如果你现在只想带走一句话，那就是：这个仓库最值得学的不是某个脚本，而是把 AE、sampler、transformer 绑在一起的契约。
记住什么： 这张地图的任务不是替代主课，而是告诉你“从哪里进、为什么这样进”。 为什么重要： 如果入口页没有把阅读路线和证据边界讲清楚，读者很容易在第一步就把产品家族、仓库身份和研究路线混成一团。 可见证据： 当前路线设计同时参考了公开资料、本地实现核对，以及多轮读者阅读后的误解集中点。 边界： 这里的三条路线是课程设计，不是官方仓库原生章节。
学习节奏
先建立总图，再深入机制，最后迁移到研究准备
目标读者
中文研究型读者；源码耐受低，但目标高
课程原则
对象先于文件树，证据边界先于判断
最终结果
能选研究起点、识别契约、理解仓库边界

三条阅读路线

你不必一次读完全部内容。按你的当前任务进入最合适的路线。

路线 A：先建立全局图

适合：刚进入项目、容易被细节压垮的读者。

顺序：0003 入口总课 -> 0001 Latent Contract -> 0002 训练缺口 -> 0004 单决策页

路线 B：深入机制层

适合：你已经知道目标是 VAE / 训练研究，想先搞清仓库运行机制。

顺序：0005 CLI 主流程 -> 0006 四大对象 -> 0007 AE 与 latent -> 0008 sampler -> 0009 transformer / KV

路线 C：迁移到研究决策

适合：你已经理解仓库大致结构，准备决定从哪个模型和哪种 fork 路线开始。

顺序：0010 模型家族与许可 -> 0011 证据边界 -> 0012 迁移与缺口页 -> 研究路线决策树

下一步： 如果你不确定从哪里开始，默认先读 0003。

课程对象，而不是文件树

本课程按教学对象组织，而不是按仓库目录组织。这样做的原因是：新读者真正要掌握的是“对象、契约、流程、边界和迁移”，不是背文件路径。

仓库身份
它是什么、它不是什么、为什么不能把 README 误读成训练系统说明。
模型家族
`klein`、`Base`、`dev` 与官方产品家族的关系。
四大对象
text encoder、autoencoder、sampler、transformer 的职责边界。
共享契约
latent 通道数、缩放倍率、token 组织、sampler 合约。
运行过程
从 prompt 到 output image 的完整闭环。
研究迁移
训练缺口、起点选择、fork 路线。
