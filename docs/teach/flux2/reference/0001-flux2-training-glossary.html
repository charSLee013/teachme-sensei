核心术语

Latent contract

autoencoder 输出、sampler 的噪声张量，以及 transformer 期望输入形状之间的固定协议。

In-channels = 128

这个仓库里所有已发布的 FLUX.2 模型都假定 latent token 的通道数是 128。你一旦改掉它，就立刻离开“只改 AE”范畴。

16 的倍数规则

输入图片会先中心裁剪到宽高都能被 16 整除，CLI 也按 height // 16 和 width // 16 分配 latent 噪声。

Distilled 模型

指推理默认值本身已经成为契约的一类 checkpoint。这个仓库里的 distilled klein 变体把 guidance=1.0 和 num_steps=4 固定住了。

Base 模型

指非 distilled 的 checkpoint，仍然暴露出更接近训练语境的 50-step 路径。在这个仓库里，它们是未来做微调或表示实验时最干净的起点。

Reference tokens

一张或多张输入参考图编码后的 latent token，在图像编辑时会拼接到待生成的 latent 序列里。

KV cache 路径

flux.2-klein-9b-kv 的特殊图像编辑路径：在第一步去噪后，把 reference token 的 attention key/value 缓存起来，后续复用。

文本编码器分裂

flux.2-dev 用 Mistral embedding，klein 模型用 Qwen3 embedding。如果你的研究会动到文本侧契约，这个分裂很重要。
