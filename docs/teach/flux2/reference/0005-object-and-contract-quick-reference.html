对象与契约速查

这页给你一个快速回忆卡：四大对象是什么、最硬的契约是什么、最常见的失效面是什么。

四大对象

Text Encoder

把 prompt 编成条件表示。关键提醒：`dev` 与 `klein` 路线不共享同一文本模型。

AutoEncoder

像素与 latent 往返。关键提醒：128 通道不是任意数字。

Sampler

组织时间步、token、CFG、多参考路径。关键提醒：Base 与 distilled 在这里直接分叉。

Transformer

执行主干去噪。关键提醒：双流 / 单流与 KV cache 是结构差异，不是小参数。

最硬契约

  • in_channels = 128
  • 像素到 latent 的 /16 网格假设
  • 图像 token 的 t / h / w / l 位置 ID 组织
  • `klein` distilled 的固定 steps / guidance 合约

最常见失效面

  • 把 AE 改动误判成纯局部改动
  • 把产品 docs 能力上限直接等同于本地实现
  • 把 9B / 32B 理解成完整系统总参数量
  • 把仓库没公开训练栈误读成官方没有训练系统
