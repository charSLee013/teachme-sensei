研究路线决策树

这页帮助你快速决定该从哪条研究路线起步。它更接近 0004 的一页式操作化版本，用来先做第一次分流，而不是替代 0012 的迁移动作页。

记住什么：这是一页决策辅助，不是官方唯一推荐路径。它把仓库真相、模型家族定位和工程摩擦压缩成可执行问题。
为什么重要：没有稳定的决策树，后面的实验方向很容易因为“看起来都能做”而漂移。
可见证据：这页的推荐顺序建立在三类事实之上：当前仓库的 latent 契约、本地注册表里的 Base / distilled 分流、以及官方 docs 对 Base 路线的 research / fine-tuning 定位。边界：具体推荐哪条路是工程迁移建议，不是官方产品页原句。置信度：中高。

第一问：你要不要先保住 latent contract？

要

优先走不破契约路线，首选 `klein-base-4b`。适合验证 VAE 内部改动、loss、轻量微调方向。

不确定

默认仍先保契约。因为这能最大化把问题限制在表示层，而不是一下子变成系统级 fork。

不要

你已经明确要改 latent 通道数、空间缩放或 token 组织。准备同时回看 AE、sampler、transformer。

第二问：你更在意什么？

  • 最低摩擦：`klein-base-4b`
  • 更高质量起点：`klein-base-9b`
  • 更接近重型研究路线：`flux.2-dev`

第三问：你此刻缺的是什么？

  • 如果缺的是“仓库真相”，回去读 `0005` 到 `0009`。
  • 如果缺的是“训练栈骨架”，补训练缺口清单与外部工程方案。
  • 如果缺的是“许可与公开边界”，回去读 `0010` 与 `0011`。
