课程 0011

证据边界

这一课不是额外的学术礼仪，而是防止你在研究准备阶段被“看起来像事实”的句子误导。我们要明确区分：官方公开事实、本地实现事实、课程重构、工程迁移判断和未知。

课程位置：这是研究迁移前的安全阀。读完它，后面很多判断你才不会拿错证据层。

记住什么

  • “有引用”不等于“就是源文事实”。
  • 当前课程里最容易被误读的，是“参数量口径”“训练缺口判断”“产品能力上限与本地实现上限的关系”。
  • 凡是会影响你选研究路线、改系统接口、或相信某个能力上限的结论，都必须先问：这到底是哪一层证据？

五层证据

官方公开事实

直接来自官方 README、官方 docs、公开 model card、公开元数据接口。

本地实现事实

当前仓库源码直接写出来的行为与结构，比如 `in_channels=128` 或 `use_kv_cache=True`。

课程重构

为了让仓库更可学而做的结构化表达，比如“四大对象”“对象图”。

工程迁移判断

把仓库理解迁移到研究或系统 fork 上的判断，比如“这个改动会联动 AE / sampler / transformer”。

未知

公开证据不足、gated 正文不可直接验证、或产品叙事无法直接落到本地实现上的部分。

可见证据：这一课里的强判断都先分清“公开资料直接支持了什么”“当前仓库直接写了什么”“哪些只是课程为了便于理解做的重构”。边界：学生页只展示简化后的证据分层，真正细的实现锚点放在下方附录。

最容易犯错的三个地方

  1. 把 9B / 32B 当成全系统总参数量。
    这常常只是主干名义规模，系统里还可能有独立文本编码器。
  2. 把“仓库没公开训练栈”说成“官方没有训练系统”。
    前者是开源实现边界，后者是无证据推断。
  3. 把产品文档的能力上限直接套到本地代码。
    比如 API 文档里“最多 10 张参考图”，不等于本地开源实现公开承诺相同上限。

练习与检索

  1. 判断下列句子属于哪层证据：“当前仓库里 `klein-9b-kv` 的模型注册表启用了 `use_kv_cache=True`。”
  2. 判断另一个句子：“如果你改 latent 通道数，研究计划就不再只是局部 VAE 实验。”
  3. 再判断：“`FLUX.2 [dev]` 完整系统总参数一定等于 32B。”为什么这句话不能直接成立？
