课程 0012

从仓库理解到训练 fork

读到这里，默认前提已经不是“我要不要选这条路”，而是“我已经选了大方向，现在该把它翻译成训练缺口、迁移动作和停止条件”。

课程位置：这是把仓库理解过渡到研究迁移的收口页。默认前提是：你已经在 0004 或 0006 做过第一次路线选择。

记住什么

  • 当前仓库最适合做“研究准备地图”，而不是直接拿来当完整训练框架。
  • 这页不负责再帮你选路，而是负责把已选路线翻译成下一步动作。
  • 真正会拉开差距的，不是路线名词本身，而是你要补哪些训练层、接受哪些新增耦合。

为什么重要

这页的重要性在于，它把前面所有“仓库理解”转成真正会改变工程行为的迁移动作。到了这里，问题不再是“选哪张路线卡更顺眼”，而是“我下一步该补什么、哪里会立刻升级成系统级工程”。

可见证据：这里的迁移建议建立在三层事实之上：当前仓库没有公开训练栈、本地实现对 latent 契约的耦合很强、官方资料把 Base 路线定位为 fine-tuning / research。边界：路线 A / B / C 仍是课程为研究准备做的重构，不是官方原始分类。置信度：中高。

你已经选了路，现在只看迁移差异

默认前提：如果你还没做第一次路线选择，先回到 0004。这页只负责把已选路线转成缺口清单和动作，不负责重新说服你选哪条路线。
当前路线 你保持什么不动 你立刻新增什么耦合 下一步最该补什么
路线 A：先保契约 保持 128 通道、/16 缩放、现有 token 组织与主干接口。 主要还是表示层与轻量训练准备，不会立刻把整个系统一起拖下水。 先补实验骨架、轻量 loss 设计、最小验证闭环。
路线 B：小心破契约 你不再承诺维持完整 latent contract。 AE、sampler、transformer 的接口联动会立刻出现，形状与 token 组织都要回看。 先把联动面列清楚，再决定是否值得继续扩大改动。
路线 C：训练向完整 fork 不再把当前仓库当成训练框架本体。 数据、loss、优化器、评估、checkpoint 生命周期都要自己建。 先设计训练栈骨架，再决定模型起点是 9B 还是 dev。
停止条件：如果你还说不清 latent contract 是什么，或者还分不清 Base 与 distilled 的差异，先不要直接跳路线 C。

仓库外必须补齐的层

数据管线

数据读取、清洗、混合、caption / reference 组织。

训练目标

loss、采样训练目标、是否有额外感知或对齐目标。

优化器与调度

optimizer、lr schedule、梯度累积、EMA 等。

评估与验证

不只是能跑，还要有可重复的质量、对齐和稳定性评估。

实验编排

checkpoint、resume、日志、版本化和结果追踪。

安全与许可边界

尤其是 gated / non-commercial 路线下的使用约束和发布边界。

可见证据：这些缺口来自对当前仓库结构、README 定位和现有工作区研究记录的综合判断。边界：这表示“当前公开开源实现没有给你这些层”，不表示官方内部不存在相应系统。

最稳的第一步

  1. 先确认你刚才选中的路线，到底要补“训练层”还是补“联动分析”。
  2. 若你现在还没有任何实验闭环，优先从不破契约的 Base 路线开始补最小训练准备。
  3. 只要一旦要改 latent 通道、缩放或 token 组织，就立刻重新审视 AE、sampler 和 transformer 的接口联动。
下一步：继续拿这页的缺口表去对照 0003 训练缺口清单；如果你想重新确认选路，回到 0006 决策树。
