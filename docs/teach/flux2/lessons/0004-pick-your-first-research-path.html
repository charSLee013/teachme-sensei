课程 0004

选出你的第一条研究路径

这节课只解决一个实际决策：如果你已经知道 FLUX.2 不是训练仓库，那么你的第一站到底该是 “先稳住 VAE 契约”，还是“直接准备训练向 fork”？

课程总地图 返回总览大课 课程 0001 课程 0002 训练缺口清单
和使命的关系： 你不是为了“懂了就结束”，而是为了把后续实验路线选对。路线一旦选错，后面几周都会浪费在错误摩擦上。

记住什么

  • 这页给出的默认路线是“通常更稳”，不是“唯一正确”。
  • 路线选择首先取决于你想不想保住 latent contract，而不是只看参数量大小。
  • 一旦你决定直接打破契约，问题就会升级成系统级 fork。

为什么重要

这页的重要性不在于它告诉你一个抽象观点，而在于它会直接影响你接下来几周的工程摩擦和实验设计路径。

可见证据： 当前仓库把 Base 路线和 distilled 路线的 sampler 合约明确分开；官方 docs 与公开 model card 也把 Base 变体定位成 fine-tuning、LoRA、研究与自定义 pipeline。 边界： “第一站经常先看 klein-base-4b”是课程重构后的工程迁移建议，不是官方唯一推荐。 置信度： 中高。

这节课只拿走一个结果

在当前工作区的使命下，最稳的默认起点通常不是“立刻造 trainer”，而是： 先围绕 flux.2-klein-base-4b 做不破契约的研究准备。

路线 A：先稳住契约

保持 128 通道、保持网格关系、保持 token 组织。适合先验证你对系统边界的理解是否真的稳定。

适合现在的你：

想先做 VAE 相关研究准备，不想立刻卷入多模块联动工程。

路线 B：直接训练向 fork

接受数据、优化器、训练循环、checkpoint、验证框架都要自己补，换来更高自由度。

适合什么情况：

你已经明确要做完整训练研究，而不是先确认契约边界。

路线 C：一开始就打破契约

直接改 latent 宽度、缩放或 token 组织。代价最高，因为会把 AE、sampler、transformer 一起拉进来。

当前不推荐：

除非你已经非常清楚自己为什么必须这么做。

什么时候不该默认从 4B 起步

默认起点是 klein-base-4b，但默认值只服务于“先压低摩擦、先保住契约”的读者。 如果你的真实目标已经不是这个，就不该机械套用。

你此刻更在意什么 更自然的入口 为什么不是默认 4B
最低硬件摩擦，先保契约做研究准备 klein-base-4b 这就是本页默认情况。
接受更高算力，换更强的 base 起点 klein-base-9b 你已经不是在追求最低摩擦，而是在追求更高质量的 base 参考。
明确要看重型研究参考，而不是低摩擦入口 flux.2-dev 你的任务已经更接近重型研究路线，不再是“先稳住开局”。
边界提醒： 这里说的是“更自然的第一站”，不是“最终必须停留的模型”。只要你的目标变成更高质量参考或更完整训练路线，升级就是合理动作。

可选自测

如果你此刻最怕的是“一动 VAE 就引发全系统联动”，第一站更应该是哪条路线？

读完后你应能立刻说出的话

  • “我现在默认先走不破契约的研究准备，而不是马上造 trainer。”
  • “如果我要直接打破 latent 契约，那已经不是只改 VAE。”
  • “如果我追求的是更高质量参考或重型研究路线，就不该机械停在 4B。”
唯一下一步： 如果你已经选定“先保契约”，继续去读 0012，把这条路线转成真正的训练缺口清单。 如果你仍拿不准系统边界，回到 0003 重新走入口主线；如果你只想看一页式分流，打开 0006 决策树。
