LingBot-Video 课程地图

课程模式：repo-course + deep-paper。语气规范：研究报告式、证据化、边界清晰。

Remember Now

  • 本课程把 LingBot-Video 作为一个"论文系统 + 公开推理仓库"来学习，而不是只按文件夹顺序阅读代码。
  • 所有强结论被分为代码事实、论文事实、教学重构、工程建议和未知项。
  • 主线是从研究动机进入公开推理链路，再进入 Pipeline、Transformer、MoE、Refiner 和多卡部署。
  • 读者最终应能解释系统、运行 smoke test、定位失败点，并明确哪些结论尚未被本地验证。

全局叙事

三条学习线

使用线

从 README、docs 和 scripts 出发，建立公开推理路径：安装、模型目录、prompt JSON、negative JSON、T2I/T2V/TI2V、single/multi GPU 脚本。

机制线

解释 Pipeline 的 denoising loop、TI2V 首帧条件、single-stream Transformer、3D RoPE、MoE routing 和 Refiner latent trajectory。

系统线

解释 backend fallback、SGLang native adapter、FSDP inference sharding、Context Parallel、CFG parallel 和无权重/有权重验证策略。

章节顺序

章节核心问题完成后能力
01 System Map仓库、论文和公开运行时如何分层？复述项目主链路，并指出训练系统不在当前仓库中。
02 Public Workflow用户如何从输入到输出？区分 T2I、T2V、TI2V 的输入、输出和帧数逻辑。
03 Rewriter And Negative为什么推理前需要 structured prompt？解释两阶段 rewriter 和 delete-only auto negative。
04 Runner OrchestrationCLI 参数如何变成 pipeline 调用？从 scripts/inference.py 追踪到 runner 的 call_kwargs。
05 Pipeline Loop扩散采样循环如何运行？解释 prompt embedding、latent、scheduler、CFG 和 VAE decode。
06 TI2V Conditioning首帧如何约束生成？解释首帧作为 VLM 图像条件和 clean latent prefix 的双重身份。
07 Single Stream视觉 token 和条件 token 如何进入同一 Transformer？解释 patchify、text projection、3D RoPE 和 block 结构。
08 Sparse MoEMoE 如何提高容量并控制活跃计算？解释 router、top-k experts、grouped execution 和 FP8 path。
09 Refiner二阶段细化如何与 base generation 连接？解释低分辨率视频重读、VAE encode、t_thresh 和 clean first frame。
10 Serving部署和多卡路径解决什么瓶颈？区分 diffusers、SGLang adapter、FSDP、CP 和 CFG parallel。
11 Validation如何在不同资源条件下验证系统？设计无权重静态检查和有权重 smoke test。

证据边界

课程正文只使用已经归类的结论。仓库代码直接支持的内容称为代码事实；arXiv source 明确支持的内容称为论文事实；为了教学而组织的路径图称为教学重构；调试和维护建议称为工程迁移；缺少权重、GPU、训练数据或原始评测脚本的部分标为未知。

详细证据见 model and paper claims、file map 和 evidence boundaries。
