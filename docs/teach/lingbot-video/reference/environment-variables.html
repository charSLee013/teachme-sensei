Environment Variables Reference

本页汇总 README、脚本和代码中出现的主要环境变量。

变量用途边界
`MODEL_DIR`脚本读取的 DiT 模型根目录。必须包含对应组件，否则加载失败或 refiner 被跳过。
`DENSE_MODEL_DIR`, `MOE_MODEL_DIR`README 示例中区分 Dense/MoE 权重。脚本通常最终传入 `MODEL_DIR`。
`REWRITER_BASE_MODEL`rewriter base VLM 路径。in-process transformers backend 必需。
`REWRITER_ADAPTER`rewriter LoRA adapter 路径。step2 Map 使用。
`PYTHON_BIN`shell 脚本选择 Python 解释器。默认 `python`。
`BACKEND`脚本传给 `--backend`。默认多为 `diffusers`。
`NPROC`, `CP`, `CFG`多卡脚本的进程数、context parallel degree、CFG parallel degree。`NPROC` 应匹配并行拓扑。
`DIFFUSERS_ATTN_BACKEND`选择 Diffusers attention backend。脚本常默认 `_native_flash`。
`HF_HUB_DISABLE_PROGRESS_BARS`runner 在非 verbose 模式下默认设置为 `1`。用于压制外部库进度条，不改变模型语义。
`TOKENIZERS_PARALLELISM`runner 在非 verbose 模式下默认设置为 `false`。用于减少 tokenizer 并行 warning。
`DIFFUSERS_VERBOSITY`, `TRANSFORMERS_VERBOSITY`runner 在非 verbose 模式下默认设置为 `error`。用于收敛日志噪声。
`LINGBOT_QWEN_ATTN_IMPLEMENTATION`覆盖 Qwen3-VL `from_pretrained` 的 attention implementation，默认 `flash_attention_3`。依赖 transformers 与对应 attention kernel 可用性。
`LINGBOT_FUSED_QKV_LINEAR`为 transformer attention 启用 fused QKV linear 分支。代码级实验开关；未在脚本中默认开启。
`LINGBOT_MOE_EXPERT_BACKEND`选择 MoE expert 执行路径。例如 `grouped_mm`、`sglang_triton_fp8`。
`LINGBOT_MOE_PAD_BACKEND`选择 grouped token padding 实现。例如 `loop` 或 `vectorized`。
`LINGBOT_MOE_REORDER_BACKEND`选择 token reorder path。例如 `sort` 或 `triton_pack`。
`LINGBOT_MOE_RESTORE_BACKEND`选择 expert output restore path。例如 `scatter`、`index_add`、`triton`。
`LINGBOT_MOE_RESTORE_CHUNK_SIZE`控制 chunked scatter restore 的 chunk 大小。必须为正整数。
`LINGBOT_QUIET_PROGRESS`关闭进度输出。也可通过 `--quiet_progress` 设置。
`LINGBOT_VERBOSE_LOGS`保留更详细导入日志。默认会压制部分外部进度条和 warning。
`SGLANG_OPT_SWIGLU_CLAMP_FUSION`SGLang shim 暴露给 fused MoE runtime 的环境 flag。只有走 SGLang MoE 相关路径时才有意义。
`SGLANG_EXPERIMENTAL_LORA_OPTI`SGLang shim 暴露的 LoRA 优化实验 flag。课程不把它解释为 DiT 质量参数。
