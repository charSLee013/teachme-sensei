07 Transformer Single Stream

本章把论文中的 task-unified single-stream DiT 与本地 Transformer 实现对齐：从 attention 数学、3D RoPE 坐标构造，到 single-stream 与 dual-stream 的架构权衡。

Remember Now

  • single-stream 的核心是将视觉 latent patch tokens 与 condition tokens 投影到同一 hidden size 后放入同一序列，共享同一套 self-attention 参数。
  • 代码中的 patchify 顺序是时空网格展开：$[B,C,T,H,W]$ 视频 latent 被切成 $T \cdot \frac{H}{p} \cdot \frac{W}{p}$ 个 patch token。
  • 3D RoPE 使用时间、高度、宽度三个坐标轴；text token 与 visual token 使用非重叠的 temporal 坐标范围，避免位置混淆。
  • QK-Norm 对 query/key 做 RMSNorm，控制 attention logits 尺度，是长序列训练稳定的关键。
  • single-stream 相比 dual-stream 的效率收益来自论文论述；本地代码能验证的是结构存在和 forward 数据流。

自注意力的数学基础

Transformer 的核心是缩放点积注意力。对一个长度为 $N$ 的 token 序列，每个 token 投影出 query、key、value，注意力权重由 query-key 内积经 softmax 归一化得到：

缩放点积注意力：

$$\text{Attn}(Q,K,V) = \text{softmax}\!\left(\frac{QK^\top}{\sqrt{d_k}}\right)V$$

其中 $Q,K \in \mathbb{R}^{N \times d_k}$，$V \in \mathbb{R}^{N \times d_v}$，$\sqrt{d_k}$ 是防止内积随维度增大而饱和 softmax 的缩放因子。

在 single-stream 设计中，序列 $N = N_{\text{video}} + N_{\text{text}}$ 是视觉 token 与文本 token 的拼接总长。这意味着一次 attention 同时建模了「视觉-视觉」「文本-文本」和「视觉-文本」三类交互，无需额外的 cross-attention 层。

QK-Norm（LingBotVideoRMSNorm 作用于 q/k）：

$$\hat{q} = \frac{q}{\sqrt{\frac{1}{d}\sum_i q_i^2 + \epsilon}} \cdot \gamma, \quad \hat{k} = \frac{k}{\sqrt{\frac{1}{d}\sum_i k_i^2 + \epsilon}} \cdot \gamma$$

归一化后 $\|\hat{q}\|,\|\hat{k}\|$ 被约束在稳定范围，使 $\hat{q}\hat{k}^\top$ 的尺度不随训练漂移，避免 attention logits 爆炸导致 softmax 饱和。

Multi-Modal 3D RoPE 的数学

旋转位置编码（RoPE）通过对 query/key 施加与位置相关的旋转，使注意力内积只依赖相对位置。一维 RoPE 对位置 $m$ 的向量施加旋转矩阵 $R_m$：

一维 RoPE 的相对位置性质：

$$\langle R_m q,\; R_n k \rangle = \langle q,\; R_{n-m} k \rangle$$

旋转角频率：$\theta_j = \text{base}^{-2j/d}$，代码中 rope_theta = 256.0。

LingBot-Video 的 3D RoPE 把 head 维度切成三段，分别编码时间 $t$、高度 $h$、宽度 $w$ 坐标。每个 token 的位置是一个三元组 $(t, h, w)$。关键在于 make_joint_position_ids 如何为不同模态分配坐标：

坐标分配（代码 make_joint_position_ids）：

$$\text{text token } i:\quad (t,h,w) = (i+1,\; 0,\; 0), \quad i \in [0, L)$$ $$\text{video token }(t',h',w'):\quad (t' + L + 1,\; h',\; w')$$

其中 $L$ 是文本长度。文本占据时间轴 $[1, L]$，视频时间轴从 $L+1$ 起始，两者非重叠。

为什么非重叠坐标至关重要

若文本和视频共享同一时间坐标（例如都从 0 开始），RoPE 会让「第 3 个文本 token」与「第 3 帧视频 token」产生虚假的相对位置为零的强关联，模型无法区分「这是文本的第 3 位」还是「视频的第 3 帧」。非重叠设计让每个 token 的时空坐标全局唯一，attention 能正确解析跨模态的相对结构。文本的 $h=w=0$ 表示它没有空间展开，只有序列顺序。

设计选择：Single-Stream vs Dual-Stream

Dual-Stream（MM-DiT / CogVideoX 风格）

文本和视频各自维护独立的 attention/FFN 参数，通过 cross-attention 或 joint-attention 的部分层桥接。优势：每个模态可以有专用的表征空间；劣势：参数量近乎翻倍，且模态间信息交换受限于桥接层的位置和频率。

Single-Stream（LingBot-Video 采用）

所有模态投影到同一 hidden size 后拼接，共享同一套 self-attention 参数。优势：参数高效、跨模态交互在每一层每个 token 对之间充分发生、实现简洁（一个 attention 覆盖全部）；代价：联合序列长度 $N_{\text{video}}+N_{\text{text}}$ 更长，attention 计算量随 $O(N^2)$ 增长，对长视频需要 Context Parallel 缓解。
