Lesson 20 / Paper Writing

英文摘要与 Introduction

Introduction 要把读者从重要问题带到可验证贡献；背景知识只保留支撑这条路径的部分。

前置：研究空白产物：中英摘要 + intro练习：claim rewrite

Remember now

摘要回答四件事：问题是什么、你做了什么、发现了什么、为什么重要。每句话都要能在正文某个实验或定义中找到对应。

直觉：给读者一条下坡路

好的 introduction 让读者不需要跳跃：先说明领域需求，再指出现有理解不足，随后定义你的问题和方法，最后用贡献清单告诉读者如何验证。不要从“生成模型近年来蓬勃发展”这种无承诺的背景开始。

context -> concrete problem -> evidence gap -> research question | v method/protocol -> findings -> contributions

公式：claim 的可追踪句法

一个可检验的英文句子通常包含：

We study X under condition C, measure Y, and find Z.

把“显著提升”“更高质量”改成带对象、条件和指标的句子。摘要不能先写结论，再临时寻找实验。

代码：从实验记录生成摘要事实

finding = {
  "setting": "Raw/Turbo at fixed resolutions and seeds",
  "variables": "steps, CFG, mu",
  "metrics": ["alignment", "quality", "latency", "memory"],
  "scope": "tested prompt set and H100 environment",
}

代码为论文正文提供可追溯依据，结构化记录用于保留 scope。论文句子必须从真实记录生成，凭印象写作会扩大主张。

实验：三轮英文改写

  1. 中文写出问题、方法、发现、意义。
  2. 英文直译，不追求华丽。
  3. 删除没有证据的形容词，补充 condition、metric 和 scope。
  4. 让同学只读摘要，要求他写出实验变量；若无法写出，摘要仍太空泛。

Example

Weak：Krea 2 achieves better quality with fewer steps.

Stronger：On our fixed prompt set and H100 evaluation, the distilled Turbo checkpoint reduces denoising steps and latency while exhibiting a measurable quality-alignment trade-off relative to Raw under the tested resolutions.

Evidence and scope

Source：paper checklist；毕业实验记录是摘要 claim 的直接来源。

Teaching use：把英文写作作为 claim compression：压缩主张时保留条件和证据。

Scope：示例句不预先断言真实实验结果；正式数字必须由毕业实验填入。

Retrieval practice

  1. 摘要中的“why important”如何避免变成营销语？
  2. 为什么结果句必须带测试条件？
  3. Contribution list 中可以写“我们证明了模型普遍更好”吗？
分层答案

标准答案：意义应连接可迁移的研究/工程决策，不是夸张排名。条件决定结论范围。不能写普遍更好，除非实验覆盖和统计证据足以支持；本课程默认使用“under the tested settings”这类 scope language。

常见误解：论文说服力来自证据与措辞的匹配；强势英文无法弥补证据缺口。

Next action：把摘要中的变量和公式移入 Method，不要让方法章节重新发明问题。
