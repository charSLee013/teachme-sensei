Lesson 10 / Mechanism

CFG、Euler 与采样步数

CFG 用一次有条件预测和一次无条件预测构造更强的条件方向；它提高控制力，也增加计算和过度引导的风险。

前置：flow matching产物：CFG 实验表实验：向量几何 + sampler

Remember now

Krea 2 采用 cond + guidance * (cond - uncond)。这里的 guidance=0 会跳过 CFG；它不是传统“scale=1 才等于无引导”的同一命名约定。

直觉：把条件差异放大

无条件预测告诉模型“当前 latent 自然会往哪里走”，有条件预测告诉它“加入 prompt 后往哪里走”。两者的差值近似是文本条件带来的方向。CFG 沿这个差值继续走，但 guidance 太大可能过饱和、过锐化或牺牲多样性。

uncond -----> natural direction | +---- cond - uncond ----> prompt-specific direction | cond + g * (cond - uncond) ---> guided velocity

公式：条件速度的外推

设 v_c = v_θ(x,t,c)，v_u = v_θ(x,t,empty)，则：

v_cfg = v_c + g(v_c-v_u) = (1+g)v_c - g v_u

当 g=0，使用 v_c；当 g>0，沿条件差异外推。若实验报告使用别的库，要明确它的 guidance_scale 定义，不能只比较数字。

代码：两次模型调用

# sampling.py
cond = model(img=img, context=txt, t=t, pos=pos, mask=mask)
if cfg:
    uncond = model(img=img, context=untxt, t=t, pos=unpos, mask=unmask)
    v = cond + guidance * (cond - uncond)
else:
    v = cond
img = img + (tprev - tcurr) * v

CFG 让每个时间步至少多一次 MMDiT forward，也让 negative prompt 只在 guidance>0 时参与。若只改 CFG，必须固定 steps、mu、seed 和 prompt。

实验：先用二维向量理解外推

cond = torch.tensor([2., 1.])
uncond = torch.tensor([1., 1.])
for g in [0., .5, 1., 3.5, 8.]:
    print(g, cond + g * (cond - uncond))

再在真实 Krea 2 上对 Raw 使用 guidance=0、3.5、4.5、7.0，对 Turbo 使用 0 和 3.5，比较同 seed 的文本对齐、饱和度、结构错误、耗时和显存。不要只看“更像 prompt”而忽略伪影。

Evidence and scope

Source：CFG 分支、Raw/Turbo 推荐设置。

Scope：当前代码没有 CLI negative prompt 选项；课程实验可以直接调用 Python API 或另写临时 harness。

Retrieval practice

  1. 为什么 CFG 通常会让推理变慢约一倍？主干模型增加了哪些计算？
  2. 为什么 Turbo 推荐 CFG=0 和 mu=1.15？
  3. 什么情况下“更高 guidance”不能写成“质量更高”？
分层答案

标准答案：cond 和 uncond 都要各跑一次主干模型。Turbo 是针对少步和固定 shift 蒸馏的 checkpoint，文档推荐关闭 CFG 并使用固定 mu；这是训练/推理约定，不是通用规律。高 guidance 可能提高 prompt 对齐，却带来过饱和、构图僵硬、细节错误或多样性下降，因此必须报告多指标。

常见误解：CFG 不是一个“更高画质”按钮；它是条件控制与分布外推的 trade-off。
