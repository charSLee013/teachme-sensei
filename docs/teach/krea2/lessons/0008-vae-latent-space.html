Lesson 08 / Mechanism

VAE 与 latent space

Krea 2 的 MMDiT 不在 RGB 像素上工作，而在一个经过 VAE 压缩和归一化的 16 通道 latent 空间中工作。

前置：shape + RoPE产物：pixel/latent 对照实验：decode contract

Remember now

VAE 是表示转换器，不是 MMDiT。采样出来的 latent 在进入 decoder 前必须使用与训练一致的 mean/std 反归一化，否则“模型输出正常”也可能得到错误图像。

直觉：在更小的画布上作画

直接在 1024x1024 像素上建模非常昂贵。VAE 把图像编码到更小的 latent 网格，让主干网络在压缩后的空间建模，再由 decoder 还原细节。压缩带来效率，也带来信息损失和 decode 的资源成本。

image pixels latent representation (B, 3, 1024, 1024) --VAE--> (B, 16, 128, 128) | MMDiT operates here | (B, 3, 1024, 1024) <--decode-- (B, 16, 128, 128)

公式：逐通道标准化

采样网络使用的 latent 通常是归一化后的 z。解码前，代码执行：

z_decode = z * latent_std + latent_mean

其中 mean/std 都是 16 个通道的统计量，reshape 后为 (1,16,1,1,1)，与视频式 VAE 的临时时间维兼容。之后 decoder 输出约在 [-1,1]，再映射到像素区间 [0,1]。

代码：显式添加时间轴

# autoencoder.py
def decode(self, x):
    x = rearrange(x, "b c h w -> b c 1 h w")
    x = (x * self.latents_std) + self.latents_mean
    return rearrange(self.ae.decode(x).sample,
                     "b c 1 h w -> b c h w")

QwenAutoencoder.decode 是一个小接口，但隐藏了三个重要约束：latent channel=16、时间帧=1、mean/std 的 channel order 必须一致。

实验：归一化错误的可视化

z = torch.randn(1, 16, 8, 8)
mean = torch.zeros(16).view(1, 16, 1, 1)
std = torch.ones(16).view(1, 16, 1, 1) * 2

correct = z * std + mean
wrong = z + mean
print(correct.std().item(), wrong.std().item())

真实 VAE 实验中，分别使用正确和错误的 std decode 同一 latent；记录颜色、对比度、饱和度和结构变化。不要把视觉差异直接叫“模型质量”，先把它归因到 decode contract。

Scope：本课讨论当前 Krea 2 的 decode 路径；不把 VAE 训练、KL loss 和所有视频帧缓存实现冒充成仓库当前功能。

Retrieval practice

  1. 为什么 MMDiT 使用 16 个 latent channels，RGB 图像则使用 3 个通道？
  2. mean/std 为什么要按 channel 对齐？
  3. 如果 VAE decode 输出 shape 多一个时间维，为什么 wrapper 要显式 squeeze/rearrange？
分层答案

标准答案：MMDiT 操作 VAE latent，16 是 latent channel 数，不是像素色彩通道。mean/std 的第 i 项只应作用于第 i 个 latent channel；错位会改变分布。Qwen VAE 是 3D/视频兼容实现，单张图仍以 t=1 处理，wrapper 把它转换回二维图像接口。

常见误解：latent channel 不是“16 种颜色”，它们是学习到的表示维度。
