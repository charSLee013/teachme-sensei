Lesson 16 / Research Method

指标、统计与错误分析

生成图像研究既不能只看一张图，也不能把一个自动指标当成真理。指标必须与问题、样本、失败类型和人工检查一起解释。

前置：实验矩阵产物：metric card实验：指标一致性

Remember now

每个指标只回答有限问题：文本对齐、审美质量、速度和人工偏好分别测量不同对象；可泛化结论还需要采样协议与统计证据。

直觉：不同尺子量不同东西

用尺子量温度不会得到长度。CLIP 类对齐指标、aesthetic score、人工偏好、生成耗时和峰值显存分别量不同对象。论文可信度来自多种指标之间的关系和冲突；挑选最高数字会引入选择偏差。

问题候选证据必须承认的盲点
是否遵循 prompt文本图像对齐、人工逐项检查语言/文化偏差，细节遗漏
视觉质量审美模型、人评、失败率审美不是事实正确
是否更快warm latency、throughput首次 compile、batch、硬件影响
是否更省资源峰值显存、权重占用测量工具和后台负载影响

公式：均值不是全部

对同一设置的 seed 指标 m_1,...,m_n，报告：

mean = (1/n)Σm_i，sample_std = sqrt(Σ(m_i-mean)²/(n-1))

若比较两个设置，至少报告差值、样本数和不确定性。不要把五个 seed 的差异写成普遍规律；应说明 prompt 类型、分辨率和 checkpoint 范围。

代码：结果表不应丢掉 provenance

record = {
    "checkpoint": checkpoint,
    "steps": steps,
    "cfg": cfg,
    "mu": mu,
    "prompt_id": prompt_id,
    "seed": seed,
    "metric_alignment": float(alignment),
    "metric_quality": float(quality),
    "latency_ms": latency_ms,
    "peak_memory_mb": peak_memory_mb,
    "failure": None,
}

每一行结果都应该能回指到生成文件和运行配置。没有 provenance 的漂亮表格不能用于复现。

实验：错误分类而非只删除

  1. 定义结构错误：手、文字、物体数量、空间关系。
  2. 定义条件错误：prompt 遗漏、风格漂移、颜色错位。
  3. 定义系统错误：OOM、NaN、编译失败、文件写入失败。
  4. 每类统计频率，并保存代表性失败样本。

错误分析用于识别结论边界并决定下一步实验；分类和频率比情绪化描述更有用。

Evidence and scope

Source：本课的统计公式和 metric card 是研究方法解释；实际指标实现必须在 source note 中记录版本和适用范围。

Teaching use：将质量、对齐和系统性能拆成不同测量问题。

Scope：本课程不把任一自动指标宣传为人类视觉质量的完整代理。

Retrieval practice

  1. 为什么平均分高但失败率高的设置仍然可能不可用？
  2. 如果 latency 降低但文本对齐降低，你会如何写结论？
  3. 负结果应该存在哪里？
分层答案

标准答案：平均分隐藏尾部风险；部署可能更关心失败率。结论应写成 trade-off，例如“在本 prompt/seed/resolution 范围内，Turbo 减少 latency，但对齐指标下降/不确定性增加”。负结果应进入原始记录、错误分析表、论文 limitation 和补充材料，并保留其出现条件。

常见误解：没有显著差异也是结果，不是“实验失败”。
