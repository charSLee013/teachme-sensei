Lesson 26 / Review

Rebuttal 与答辩

Rebuttal 的任务是减少不确定性；答辩的任务是让别人能复述你的证据边界。

前置：objection map产物：rebuttal draft练习：90 秒答辩

Remember now

每条回应采用“感谢/澄清 -> 直接回答 -> 证据 -> 修改 -> 剩余限制”的结构，优先处理 reviewer 真正担心的 claim。

直觉：降低对方的搜索成本

审稿人没有时间重新猜你的实验。回应中直接指出表格、图、配置、补充实验和修改位置，让他能快速确认你是否处理了问题。

reviewer concern -> restate fairly -> answer in one sentence -> point to evidence (Table/Figure/appendix) -> state manuscript change -> retain or narrow scope

公式：回应的证据链

response = respect + answer + evidence + revision + boundary

数据暂时无法补充时，可以明确说明当前协议尚未建立 X，随后删除对应主张并澄清限制。这样的回应让结论重新对齐现有证据。

代码：把回复绑定到 artifact

response = {
  "reviewer_id": "R1-C2",
  "answer": "We agree that one seed is insufficient.",
  "new_experiment": "five seeds on the fixed prompt set",
  "evidence": ["results/r1_c2.json", "Table 3"],
  "manuscript_change": "Section 4.2 and limitation paragraph",
  "remaining_scope": "not tested beyond 1024px",
}

科研写作的工程化体现在：每一句 response 都能回到结果文件和论文版本。

实验：90 秒答辩

  1. 30 秒说明问题和固定研究主线。
  2. 30 秒解释方法和控制变量。
  3. 20 秒给出最重要观察和指标。
  4. 10 秒主动说出限制。

录音后检查是否使用了“always、significantly、better、general”而没有条件。把绝对词替换为 measurable scope language。

Evidence and scope

Source：第 20-25 课的 claim/evidence 记录、paper checklist。

Teaching use：将 rebuttal 变成可审计的研究沟通。

Scope：课程不模拟真实编辑决定，只训练回应证据和边界的能力。

Retrieval practice

  1. 为什么第一句不应立即攻击审稿人的前提？
  2. 没有新实验时，rebuttal 还能做什么？
  3. 什么样的回答看似礼貌但没有信息？
分层答案

标准答案：先公平复述能确认你理解问题，再给回答。没有新实验可以澄清定义、补充实现细节、提供已有证据、修改 claim 或明确 limitation。只说“Thank you, we will consider this”没有回答、证据或修改，因此信息量为零。

常见误解：rebuttal 不是另一个 introduction；它必须逐条对应 objection。
