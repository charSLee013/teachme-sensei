Lesson 23 / Paper Writing

图、caption 与视觉证据

生成模型论文的图不仅展示“好看的结果”，还要展示过程、比较、失败和读者应该观察的变量。

前置：实验表产物：pipeline figure + result grid练习：caption rewrite

Remember now

一张图必须有观察任务。Caption 要说明对象、设置、变量和读者应得出的有限结论，；单独写“Qualitative comparison”缺少这些信息。

直觉：图是带方向的观察

读者看图时会寻找模式。你要通过排列、标注和 caption 告诉他比较什么：同一 prompt 的不同 CFG？同一 seed 的不同 checkpoint？不同 resolution 的失败边界？没有方向的图会把偶然差异伪装成结论。

Figure 1: system pipeline prompt -> encoder -> text states noise -> packer -> image tokens -> MMDiT -> VAE -> output time / CFG / resolution controls Figure 2: same prompt + same seed columns = Raw baseline | Raw ablation | Turbo baseline caption states metrics, steps, cfg, resolution and seed count

公式：caption 的信息密度

一个完整 caption 至少编码：

caption = object + comparison + condition + observation + scope

例如：对象是“same prompts and seeds 的输出”，比较是“Raw/Turbo”，条件是“1024px、推荐 steps/CFG”，观察是“latency and alignment trade-off”，scope 是“测试集合和 H100 环境”。

代码：图必须可从 records 生成

plot_grid(
    images=sorted_outputs,
    rows="prompt_id",
    columns="setting",
    annotate=["seed", "steps", "cfg"],
    output="figures/qualitative-grid.png",
)

不要手动从文件夹挑图。排序规则、失败样本和选择理由都应可追踪；若使用人工挑选，论文中要说明选择协议。

实验：caption 盲读测试

  1. 让同伴只读 caption，不看正文。
  2. 让他回答图比较了什么、固定了什么、结论范围是什么。
  3. 若答不出来，补充条件或拆图。
  4. 把“看起来更好”改成可观察的描述。

Evidence and scope

Source：项目 prompting examples、实验记录和论文图表结构。

Teaching use：将真实生成图从展示素材变成受控视觉证据。

Scope：示例 caption 不包含未经实验验证的优越性结论。

Retrieval practice

  1. 为什么同一张图既不能同时证明质量提升和速度提升？
  2. caption 需要写 seed 吗？
  3. 什么时候应展示失败样本？
分层答案

标准答案：不同 claim 需要不同测量；一张图可能帮助说明多个现象，但不能替代速度表或统计质量指标。只要 seed 影响可见结果且涉及公平比较，就应写明。失败样本在研究问题涉及稳定性、边界或 trade-off 时必须展示。

常见误解：图越漂亮越能支持论文；事实上代表性选择可能造成 selection bias。
