Lesson 02 / Foundation

PyTorch 张量与 shape 思维

读深度学习代码时，shape 属于接口契约。任何维度解释错误都会让后续公式和图像失去可信度。

前置：第 1 课产物：shape contract实验：CPU 可完成

Remember now

每个张量都至少有四个问题：它的 shape 是什么？它在哪个 device？它是什么 dtype？每个 axis 的语义是什么？

直觉：维度是名词，不是数字

(B, C, H, W) 这四个维度依次描述图片数量、通道数量、行数和列数。同一个数字处在不同轴时具有不同含义。Krea 2 会在 (B,C,H,W) 的 latent 和 (B,N,D) 的 packed token 之间转换。

pixel image: (B, 3, 1024, 1024) VAE latent: (B, 16, 128, 128) compression = 8 patch tokens: (B, 4096, 64) patch = 2 text states: (B, 512, 12, 2560) 12 selected layers combined: (B, 4608, 6144) text + image

公式：reshape 不改变元素总数

若张量 x 的元素总数为 ∏ s_i，合法 reshape 必须保持这个乘积不变。对 latent 做 patchify 时：

(B, C, H, W) -> (B, H/p * W/p, C*p*p)

例如 (1,16,128,128) 在 p=2 时成为 (1,4096,64)，因为 4096*64 = 16*128*128。这不是压缩；它只是把空间邻域重新排列成 token。

代码：追踪 `prepare`

# sampling.py
b, _, h, w = img.shape
h_, w_ = h // patch, w // patch
img = rearrange(
    img,
    "b c (h ph) (w pw) -> b (h w) (c ph pw)",
    ph=patch, pw=patch,
)

这里的 h_ 与 w_ 是 token 网格尺寸；img 的最后一维保存一个 patch 内的所有通道和像素。后续 mmdit.py 的 first Linear 正好接收 channels * patch**2。

实验：用 round-trip 抓 shape 错误

import torch
from einops import rearrange

x = torch.randn(2, 16, 8, 10)
p = 2
tokens = rearrange(x, "b c (h ph) (w pw) -> b (h w) (c ph pw)", ph=p, pw=p)
y = rearrange(tokens, "b (h w) (c ph pw) -> b c (h ph) (w pw)",
              h=4, w=5, c=16, ph=p, pw=p)
assert x.shape == y.shape
assert torch.equal(x, y)

实验重点是确认转换可逆。把 h=4 改成 h=5，错误应该在当前转换处暴露，避免十层之后才表现成坏图。

Evidence and scope

Source：sampling.py:prepare、mmdit.py:first。

Teaching use：把 einops 模式读法与真实 Krea 2 token contract 连接。

Scope：本课讲布局变化，不把 patchify 误称为 VAE 压缩，也不涉及训练反向传播。

Retrieval practice

  1. 为什么 1024x1024 图像经 VAE 后不是 1024x1024 latent？
  2. 为什么 patchify 后 token 数是空间面积除以 patch**2？
  3. 如果输入宽度不是 16 的倍数，当前 sampler 做了什么？
分层答案

提示：区分 VAE 的 compression=8 和 patch=2。

标准答案：VAE 将空间降到 128x128；patchify 再把 2x2 latent 邻域打包，所以 token 网格是 64x64，共 4096 个 token，每个 token 有 16x2x2=64 个数。当前 roundup 将宽高向上补齐到 compression*patch=16 的倍数。

常见误解：patchify 不减少元素总数；真正的空间压缩发生在 VAE。
