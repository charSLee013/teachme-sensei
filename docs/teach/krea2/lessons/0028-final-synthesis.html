Lesson 28 / Synthesis

最终综合与迁移

课程终点是把这套研究方法迁移到新的生成模型、仓库和论文问题；Krea 2 常量仅服务于当前案例。

前置：毕业论文包产物：迁移审查终点：独立研究循环

Remember now

面对新模型，先重建对象地图和接口 contract，再重建机制、实验和证据。不要把 Krea 2 的参数、prompt 模板和结论复制到不满足相同条件的系统。

直觉：带走方法，不带走答案

换成新的 text encoder、VAE、scheduler 或 backbone 时，具体 shape 会变，但“对象 -> 表示 -> 过程 -> 约束 -> 验证”的阅读方法仍然适用。

new repository -> inventory and source map -> shape/interface contracts -> intuition/formula/code/experiment -> controlled question -> evidence graph -> paper claim with scope

公式：研究循环

把独立研究写成：

Observe -> Model -> Predict -> Measure -> Revise -> Communicate

如果观察和模型冲突，检查实现；如果预测和实验冲突，检查假设和变量；如果沟通超出证据，缩小 claim。这个循环比任何一个特定模型名称更值得记住。

代码：迁移时先写 contract

contract = {
    "conditioner": {"input": "text", "output": "..."},
    "latent": {"shape": "...", "normalization": "..."},
    "sampler": {"state": "...", "time_direction": "..."},
    "backbone": {"tokens": "...", "mask": "..."},
    "decoder": {"dtype": "...", "output_range": "..."},
}

先填未知项，不要猜。未知项会变成下一轮 source search 或实验；这就是从初学者向研究作者升级的核心行为。

实验：迁移审查

  1. 选择一个新 diffusion/flow-matching 仓库。
  2. 只用 30 分钟画出模块和数据流，不运行大模型。
  3. 列出三条与 Krea 2 相同的契约和三条不同的契约。
  4. 提出一个可复现问题，写出固定项、变化项和 scope。
  5. 用 90 秒向别人讲清楚你知道什么、尚不知道什么。

迁移审查要求把事实、假设和待验证问题分开，并为每项标注来源或下一步验证。

Evidence and scope

Source：全课程的 local code、官方 source notes、实验 protocol 和毕业 rubric。

Teaching use：把 Krea 2 具体知识抽象成可迁移的科研工作流。

Scope：迁移到新仓库时必须重新核验版本、license、模型配置和实验条件。

Retrieval practice

  1. 新模型没有 README 时，你的第一步是什么？
  2. 哪些 Krea 2 结论不能直接迁移？
  3. 什么行为最能证明你已经不再只是“会跟着教程跑”？
分层答案

标准答案：先建立文件/入口/依赖/对象/来源地图。具体的 34/5 token 常量、16 latent channels、mu 推荐值、CUDNN backend 和质量结论都不能无条件迁移。能够提出受控问题、建立 contract、记录失败、解释证据范围并写出可复核结论，才是独立研究能力。

常见误解：独立研究能力表现为提出可验证的问题、检查答案，并对采用的帮助保留证据记录。

毕业动作：回到 课程地图，打开你的论文包、实验 manifest 和 objection map，做一次完整的自审与答辩。
