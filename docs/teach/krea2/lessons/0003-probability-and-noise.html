Lesson 03 / Foundation

概率、噪声与可复现随机性

生成模型从随机 latent 出发，再由条件和模型逐步更新状态并形成图像。

前置：shape产物：seed protocol实验：CPU 可完成

Remember now

seed 不是图片本身，也不是质量参数。它只固定随机过程的起点；相同 seed 只有在模型、版本、设备、dtype、参数和算法都相同的条件下才有可比意义。

直觉：从同一堆沙子开始

把 latent noise 想成一堆随机沙子。prompt 不会直接“画出”对象，它改变的是每个时间点应该朝哪个方向移动。换一个 seed，就像换一堆沙子；换 CFG 或 steps，则像换了雕刻规则。

seed=7 -> noise A -> trajectory A -> image A seed=8 -> noise B -> trajectory B -> image B same seed + different CFG/steps/model version -> different trajectory -> not a controlled seed comparison

公式：随机变量与条件分布

初始 latent 可写成 X_1 ~ N(0, I)。模型要近似的是给定文本条件 c 后的生成过程，最终得到 X_0 ~ p(x | c)。实验中我们常用一个有限 seed 集合 S={s_1,...,s_k} 估计平均表现和方差，而不能只看一张最好的图。

若评价指标为 m(s)，平均值是 mean(m(s))，稳定性至少要同时报告 std(m(s)) 或置信区间。

代码：每张图独立生成器

# sampling.py
torch.randn(
    1, ae.channels, height // ae.compression,
    width // ae.compression,
    device=device, dtype=dtype,
    generator=torch.Generator(device=device).manual_seed(seed + i),
)

这里每个 image index 使用 seed+i，所以 batch 顺序不会把一张图的随机数消耗掉后影响下一张。这是一个明确的 reproducibility 选择，但跨 Torch/CUDA 版本仍不保证 bitwise identical。

实验：seed、参数与方差

  1. 用 toy denoiser 对 seeds 0-4 生成向量。
  2. 固定 seeds，只改一个变量，例如 steps 8 -> 16。
  3. 记录均值、标准差和失败样本，不只保存视觉最好的输出。
results = []
for seed in range(5):
    image = toy_sample(seed=seed, steps=8, guidance=0.0)
    results.append(metric(image))
print(sum(results) / len(results))

Evidence and scope

Source：sampling.py 的 seeded noise；概率分布与均值/方差是课程基础解释。

Teaching use：把“随机生成”转成可设计的统计实验。

Scope：本课不声称一个 seed 能代表模型整体质量；真实质量评估需要固定 prompt 集合、多个 seed 和指标。

Retrieval practice

  1. 为什么改变 seed 后不应把两张图的差异归因于 CFG？
  2. 如果只报告一张 seed=0 的图，读者无法知道什么？
  3. 如何设计一个只测 steps 影响的公平实验？
分层答案

标准答案：seed 同时改变初始随机 latent 和整条轨迹，因此它是混杂变量。单张图无法估计平均质量、方差或失败率。公平设计应固定 checkpoint、prompt、resolution、CFG、mu、dtype、device 和 seed 集合，只改变 steps，并报告均值、离散程度和代表性图像。

常见误解：固定 seed 只解决随机起点，不解决版本、kernel、浮点数和模型 revision 的差异。
