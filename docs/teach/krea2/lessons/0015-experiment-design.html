Lesson 15 / Research Method

研究问题、变量与实验矩阵

论文实验用最小的受控变化回答明确问题；参数组合和截图只有进入完整协议后才构成证据。

前置：hardware matrix产物：预注册实验表实验：pilot study

Remember now

先写“如果 X 改变，我预计 Y 如何改变，为什么”，再写代码。没有可证伪的预期，实验就只能产生素材，不能产生证据。

直觉：从旋钮到问题

steps、CFG、mu、resolution、Raw/Turbo 都像旋钮，但研究不是测旋钮本身。问题必须包含对象、变化、结果和条件，例如：“在固定 prompt、seed 和硬件下，Raw 的 resolution-aware shift 是否改善文本对齐/质量效率比？”

question -> hypothesis -> independent variable -> controlled baseline | +-> metrics + seeds + failure policy | result -> interpretation -> claim scope

公式：因果比较的最小结构

设因素为 X，结果为 Y，控制变量为 C。一个可解释的比较要求近似：

ΔY = Y(X_1, C) - Y(X_0, C)

如果同时改变 checkpoint、steps、CFG 和 seed，那么观测到的 ΔY 无法归因于单一因素。交互项也要预先设计，并按记录解释。

代码：把矩阵变成配置

experiments = [
    {"checkpoint": "oss_raw", "steps": 52, "cfg": 3.5, "mu": None},
    {"checkpoint": "oss_raw", "steps": 28, "cfg": 4.5, "mu": None},
    {"checkpoint": "oss_turbo", "steps": 8, "cfg": 0.0, "mu": 1.15},
]
for config in experiments:
    run(config, prompts=PROMPTS, seeds=SEEDS,
        width=1024, height=1024, dtype=torch.bfloat16)

配置应保存为结构化数据，命令由配置生成。每次运行保存完整 config、环境和结果路径，让论文表格可以从记录重建。

实验：先做小 pilot

  1. 选 6 个 prompt：照片、插画、文字、复杂构图、人物和非英语风险案例。
  2. 每个设置跑 2 个 seed，检查指标是否有区分度。
  3. 发现指标不稳定或成本过高时，修改方案并记录修改理由。
  4. pilot 通过后再扩大到正式 seed 集合。

pilot 不是偷看答案后删掉失败结果；它是设计验证，必须保留记录。

Evidence and scope

Source：课程固定毕业问题、CLI 参数。

Teaching use：把 Krea 2 参数表转换为一个可解释的实验设计。

Scope：本课不承诺这些变量存在因果关系；实验只是让关系可被公平检验。

Retrieval practice

  1. 为什么“把所有参数都试一遍”不是一个好研究问题？
  2. 什么是控制变量？在毕业实验中至少列出五个。
  3. pilot 结果能不能直接写成最终结论？
分层答案

标准答案：全组合会产生大量比较和多重解释，缺少中心问题。控制变量应包括 checkpoint、prompt 集合、seed、resolution、dtype、GPU、模型 revision 等。pilot 用于验证设计和指标，不足以支撑最终结论；正式实验需要扩大样本、预先定义分析规则并报告不确定性。

常见误解：证据强度主要来自变量控制、重复测量和清楚的解释边界，实验数量只是覆盖范围的一部分。
