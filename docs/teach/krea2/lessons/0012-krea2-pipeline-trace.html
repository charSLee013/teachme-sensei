Krea 2 完整 pipeline trace

现在把前十一课压缩成一次可追踪的运行：每一步都写出状态、接口和可验证问题。

前置：第 1-11 课产物：端到端 trace sheet实验：dummy pipeline

Remember now

读 pipeline 时沿状态变量追踪转换；文件名用于定位实现。一次采样中最重要的状态是 prompt representation、latent tokens、time、velocity 和 decoded image。

直觉：一次运行的状态机

S0 prompt string -> S1 text hidden states + text mask -> S2 Gaussian latent + image positions -> S3 packed image tokens + joint mask -> S4 guided velocity at t_k -> S5 updated tokens at t_{k+1} -> repeat S4/S5 -> S6 unpatched normalized latent -> S7 pixel image

任何研究主张都应指向一个状态或状态转换。例如“CFG 更强”需要同时观察 S4 的 velocity 差异与最终 S7 指标；CLI 参数只记录配置。

公式：组合函数

将 pipeline 写成：

c, m_t = E(prompt)

x_1 = N(0,I)

p, m = P(x_1, m_t)

x_{k+1} = x_k + Δt_k G(v_θ(x_k,c,t_k), v_θ(x_k,c_empty,t_k))

image = D(U(x_0))

其中 E 是文本编码，P 是 patch/position/mask 准备，G 是 CFG，U 是 unpatchify，D 是 VAE decode。

代码：入口与边界

# inference.py
dit, ae, encoder = _pipeline(checkpoint=checkpoint)
images = sample(
    dit, ae, encoder, [prompt] * num_images,
    width=width, height=height, steps=steps,
    guidance=cfg, seed=seed, y1=y1, y2=y2, mu=mu,
)
for i, image in enumerate(images):
    image.save(f"{output}_{i}.png")

CLI 是外部接口；sample 是更好的测试 seam，因为它接收 model、ae、encoder，可以使用 dummy adapter。研究时先在 seam 上验证，再触碰真实权重。

实验：dummy pipeline 的端到端检查

  1. 用 dummy encoder 返回固定 hidden states/mask。
  2. 用 dummy model 返回零 velocity，验证 steps 循环和 shape。
  3. 用 dummy VAE 放大 latent，验证图像尺寸和 padding 规则。
  4. 分别打开 CFG=0 和 CFG>0，数 model call 次数。
assert len(images) == num_images
assert cond_calls == steps
assert uncond_calls == (steps if guidance > 0 else 0)

零 velocity 的输出不会是好图，但它能证明控制流。把“控制流正确”和“模型质量正确”分成两类测试，是研究工程的基本纪律。

Evidence and scope

Source：CLI main、sample。

Teaching use：把多个文件重构成可复用的状态机和函数组合。

Scope：本 trace 描述当前 open inference code，不声称涵盖 ComfyUI、SGLang 或 hosted API 的额外行为。

Retrieval practice

  1. 哪一个状态最适合研究 CFG 对 velocity 的影响？
  2. 为什么 dummy model 能验证控制流但不能证明图像质量？
  3. 哪个函数是最自然的测试 seam？为什么？
分层答案

标准答案：CFG 直接改变每个时间步的 S4 guided velocity；dummy model 能验证参数传递、调用次数、shape 和输出，但它没有训练后的语义能力，不能证明视觉质量。sample(model, ae, encoder,...) 是自然 seam，因为三大模块可以替换成小 adapter 而不改采样控制流。

常见误解：端到端运行成功只证明接口接通，不证明研究结论成立。
