Lesson 05 / Foundation

Token、embedding 与条件输入

模型不直接接收“一个自然语言句子”。它接收 token id，再把 id 映射到能参与矩阵运算的向量，最后形成多层 hidden states。

前置：推理函数产物：prompt contract实验：真实 tokenizer + tiny model

Remember now

tokenizer 决定文字如何切分，embedding 决定 token 如何进入向量空间，hidden state 记录经过多层上下文混合后的表示，mask 决定哪些位置参与注意力。

直觉：把句子变成带位置的积木

“a fox walking in the snow” 对人是一个整体，对模型先是一串整数。特殊 token 负责消息边界，padding 让 batch 中不同长度对齐，assistant suffix 让模板保持训练时的对话布局。数字本身没有语义，语义来自 embedding 参数和层间变换。

raw prompt -> chat template prefix + prompt + padded region + assistant suffix -> token ids: [151644, 8948, ..., 151645, ...] -> embeddings: (B, L, 2560) -> selected hidden states: (B, L, 12, 2560) -> valid mask: (B, L)

公式：embedding 与层选择

给定词表大小 V 和 hidden size d，embedding 矩阵是 E ∈ R^{V×d}。token id i 取出 E_i。经过第 l 层得到 H^{(l)} ∈ R^{B×L×d}。Krea 2 不只使用最后一层，而是选择：

C = stack(H^{(2)}, H^{(5)}, ..., H^{(35)}, axis=2)

所以条件张量是 (B,L,12,d)；之后 MMDiT 的 text fusion 将 layer axis 投影成一个文本向量。

代码：模板和固定长度

# encoder.py
text = [self.prompt_template_encode_prefix + item for item in text]
inputs = self.tokenizer(
    text, truncation=True, padding="max_length",
    max_length=self.max_length + prefix_idx - suffix_start_idx,
    return_tensors="pt",
)
suffix_inputs = self.processor(text=suffix_text, return_tensors="pt")
input_ids = torch.cat([inputs["input_ids"], suffix_ids], dim=1)
mask = torch.cat([inputs["attention_mask"].bool(), suffix_mask], dim=1)

当前 Qwen tokenizer 实测 prefix 为 34 token、suffix 为 5 token，因此截掉 prefix 后得到固定长度 512。这个常量不是装饰：一旦 token 数改变，hidden-state 和 MMDiT 预期就会错位。

tokens = tokenizer(["red square", "a long prompt ..."],

Retrieval practice

  1. 为什么 padding 位置仍会出现在 hidden-state 张量里？
  2. Krea 2 选择多层 hidden states 会带来哪些信息？
  3. prefix 被切掉后，为什么 suffix 不能随意追加在另一处？
分层答案

标准答案：batch 需要固定 L，所以 padding 被计算出来但由 mask 标记为无效。多层选择把不同抽象深度的语言特征交给 text fusion，projector 学习如何组合它们。suffix 是训练布局的一部分，改变它的位置会改变有效位置、因果关系和位置编码，不能只把字符串内容看作等价。

常见误解：mask 不是删除 tensor；它是在注意力计算中限制可见性。
