Lesson 07 / Mechanism

RoPE 与三轴位置

图像 token 不是一排没有空间的词。Krea 2 用三轴位置把文本、latent 行和 latent 列放进同一条 sequence。

前置：attention产物：position map实验：旋转矩阵

Remember now

RoPE 不给 token 加一个独立的位置向量，而是按位置旋转 Q/K，使它们的相对位置关系进入点积。

直觉：同一句话与同一张图的坐标

文本 token 可以都放在“文本轴原点”，图像 token 需要知道自己位于哪一行哪一列。Krea 2 的 pos 形状是 (B,L,3)：第 0 轴代表时间/文本共享轴，第 1 轴代表 latent 行，第 2 轴代表 latent 列。

text text text text ... -> (0, 0, 0) image grid token (row=0,col=0) -> (0, 0, 0) image grid token (row=0,col=1) -> (0, 0, 1) image grid token (row=1,col=0) -> (0, 1, 0) image grid token (row=1,col=1) -> (0, 1, 1)

公式：二维旋转块

对每一对通道，位置 p 的旋转为：

R(θp) = [[cos(θp), -sin(θp)], [sin(θp), cos(θp)]]

频率 ω_i = θ^{-2i/d}，旋转后的 query/key 使点积依赖相对位置。Krea 2 将 head dimension 128 分成 [32,48,48] 三个偶数轴段，并分别对三轴计算频率。

代码：position -> freqs -> q/k

实验：旋转保留长度，改变方向

p = torch.tensor([[1., 0.]])
q = torch.tensor([[1., 0.]])
theta = 3.1415926 / 2
R = torch.tensor([[torch.cos(theta), -torch.sin(theta)],
                  [torch.sin(theta),  torch.cos(theta)]])
rotated = q @ R.T
print(rotated)  # 约等于 [0, 1]
print(torch.linalg.vector_norm(q), torch.linalg.vector_norm(rotated))

在 toy 实验中改变 token 的行坐标而不改变内容，观察 attention logits 如何变化。再交换两个图像 token 的列坐标，验证模型收到的是不同空间布局。

Evidence and scope

Source：mmdit.py:rope/ropeapply、sampling.py:prepare。

Teaching use：把位置编码从抽象公式连接到 image grid。

Scope：这里的三轴是 Krea 2 MMDiT 的 position contract；Qwen3-VL 内部文本位置处理另有 tokenizer/mRoPE 约束，后续单独讨论。

Retrieval practice

  1. 为什么文本 token 可以共用零坐标，而图像 token 不能？
  2. 为什么 axes 的维度必须是偶数？
  3. 如果 patchify 顺序和 position grid 顺序不一致，会出现什么问题？
分层答案

标准答案：文本在当前 MMDiT 中共享同一个文本位置设计，图像需要二维空间关系。旋转以二维通道对为单位，所以每个轴段需要能拆成偶数。若 token 排列与位置坐标错位，内容 A 会带着位置 B 进入模型，输出可能仍然有数值却失去空间语义。

常见误解：RoPE 不会自动修复错误的 token 顺序；它只能编码你提供给它的坐标。
