Lesson 24 / Paper Writing

限制、负结果与诚实结论

高质量研究写作主动说明限制结论的条件，并把失败整理成下一位研究者可复用的信息。

前置：图表与统计产物：limitations section练习：claim narrowing

Remember now

限制定义结论的边界函数：它明确观察成立的 prompt、分辨率、硬件、版本、指标和 seed 范围。未来工作应从这些边界出发。

直觉：给结论画边框

一张地图没有边界会误导使用者。实验结论同样需要边框：你测了什么，没有测什么，哪个变量可能改变结果，哪个失败暂时无法解释。

claim: Turbo reduces latency under tested settings boundary 1: prompt set boundary 2: resolution boundary 3: recommended steps/cfg/mu boundary 4: H100 + software revision unknown: other hardware, languages, unseen prompt types

公式：结论的条件化

不要写：

Turbo is better.

写成：

Under C, Turbo improves Y1 while changing Y2, with the observation supported by S samples and M metrics.

如果实验没有测质量，就不能在速度结论后顺手加“without quality loss”。这是从 observation 跳到 unsupported claim。

代码：失败也要成为 record

try:
    image = sample(...)
    record["status"] = "ok"
except Exception as exc:
    record["status"] = "failed"
    record["error_type"] = type(exc).__name__
    record["error"] = str(exc)
    save_record(record)

当前项目的实际 sampler 对 steps、dtype、device、负 prompt 长度等输入缺少边界校验；课程将这些观察写进 limitation 和工程改进，并明确当前可用设置的证据范围。

实验：负结果三问

  1. 失败是否由实现错误、资源不足、指标不合适还是模型能力造成？
  2. 这个失败是否可重复？
  3. 它会缩小哪条 claim 的范围，或提出哪条新问题？

不要这样写

“由于时间有限，我们没有测试其他分辨率，但模型应该同样有效。”

更好的写法是：“本研究未测量其他分辨率，因此不对该范围做结论；分辨率外推是后续实验。”

Evidence and scope

Source：实验与复现来源、实际运行记录。

Teaching use：训练 claim narrowing 和负结果记录。

Scope：限制的具体内容要由真实实验产生；本页给出方法和示例，不预设 Krea 2 的最终结果。

Retrieval practice

  1. 为什么“未来工作”不能替代当前 limitation？
  2. 一个 OOM 能否证明模型质量差？
  3. 当指标和人工判断冲突时，结论应如何写？
分层答案

标准答案：未来工作没有说明当前结论的有效范围。OOM 首先证明当前资源/规模配置不可用，不能直接推出质量差。冲突应被报告，并解释指标 scope、人工协议和可能偏差；只保留有利结果会产生选择偏差。

常见误解：诚实报告限制不会降低论文质量，隐藏限制才会损害可信度。
