Lesson 09 / Mechanism

Diffusion、噪声路径与 flow matching

生成过程学习一条从随机状态走向数据状态的路径，图像是整条轨迹结束后的结果。

前置：latent + 概率产物：时间路径图实验：线性 flow toy

Remember now

在 flow matching 的视角下，模型预测的是速度场 v_θ(x_t,t,c)；采样器把速度乘以时间差，重复积分得到下一状态。

直觉：给噪声一个方向

从噪声到图像不是一条预先画好的直线，因为不同 prompt、不同当前位置和不同时间需要不同方向。模型的任务是：看到当前状态、条件和时间，回答“现在往哪里走”。

t=1.0 noise -----> partially organized -----> t=0.0 image v_theta(x_t, t, c) changes with state, time and text Euler step: x_next = x_current + (t_next - t_current) * velocity

公式：路径与速度

设数据样本为 x_0，噪声为 x_1，最简单的线性插值是：

x_t = (1-t)x_0 + t x_1

其目标速度是：

dx_t/dt = x_1 - x_0

实际模型学习条件速度场，使许多样本路径在统计上指向数据分布。推理时使用离散网格 t_0,...,t_K，Euler 更新为：

x_{k+1} = x_k + (t_{k+1}-t_k)v_θ(x_k,t_k,c)

代码：Krea 2 的时间 grid

# sampling.py
ts = torch.linspace(1, 0, steps + 1)
if mu is None:
    slope = (y2 - y1) / (x2 - x1)
    mu = slope * seq_len + (y1 - slope * x1)
ts = math.exp(mu) / (math.exp(mu) + (1.0 / ts - 1.0) ** sigma)

for tcurr, tprev in zip(ts[:-1], ts[1:]):
    v = model(img=img, context=txt, t=t, pos=pos, mask=mask)
    img = img + (tprev - tcurr) * v

注意时间从 1 走向 0，因此 tprev-tcurr 是负数。模型预测的是速度，代码并不是简单地“减去噪声”。

实验：线性 flow 的 Euler 误差

x0, x1 = torch.tensor(0.), torch.tensor(10.)
def exact(t): return (1-t)*x0 + t*x1
def velocity(t): return x1 - x0

for steps in [1, 2, 4, 8]:
    ts = torch.linspace(1, 0, steps + 1)
    x = x1.clone()
    for a, b in zip(ts[:-1], ts[1:]):
        x = x + (b-a) * velocity(a)
    print(steps, x.item(), exact(torch.tensor(0.)).item())

线性例子可以精确积分；再把 velocity 改成随 x 变化的函数，观察 steps 增加如何减少离散化误差。这为后续 Raw/Turbo steps 实验建立“数值误差 vs 计算成本”的直觉。

Evidence and scope

Source：timesteps、Euler loop；理论来源见 flow matching source note。

Teaching use：把连续时间模型重构为初学者可模拟的离散状态转移。

Scope：课程解释 Krea 2 当前 sampler 的 flow-matching 视角，不推断其训练数据或内部损失实现。

Retrieval practice

  1. 为什么同一个 steps 数不代表同一个计算轨迹？
  2. mu 改变的是模型参数还是时间点？
  3. 为什么 steps 越多通常更慢，但不保证视觉质量必然更高？
分层答案

标准答案：steps 只是网格数量，时间 shift、模型、CFG、seed 和分辨率共同决定轨迹。mu 改变时间 grid，不改 checkpoint 参数。更多 steps 减少数值离散误差但增加 forward 次数，过多步骤也可能偏离 distilled checkpoint 的训练分布。

常见误解：Turbo 的 8 steps 不是 Raw 的 8 steps 只少算几次；Turbo checkpoint 的训练目标已经针对少步采样改变。
