Lesson 11 / Mechanism

DiT 与 MMDiT

Krea 2 主干先把文本与图像整理为 token，再送入联合 Transformer 流进行处理。

前置：attention + latent产物：主干 block 图实验：tiny forward

Remember now

MMDiT 的关键接口是联合序列：先融合文本条件，再把 text token 和 image token 拼接，让同一套 attention 在两个模态之间传递信息。

直觉：两种语言进入同一张桌子

文本 token 说“我要什么”，图像 token 说“当前 latent 长什么样”。如果两者永远分开，图像只能在最后被弱条件影响；联合 sequence 让图像 token 在每个 block 都能访问文本信息。

context: (B, text_len, 2560) -> txtfusion -> txtmlp -> (B, text_len, 6144) image: (B, image_len, 64) -> first -> (B, image_len, 6144) combined = [context | image] -> 28 x SingleStreamBlock -> select image part -> velocity tokens

公式：条件残差 block

每个 block 通过时间向量 t 产生调制参数。简化表示：

x' = x + gate_a · Attention((1+scale_a)Norm(x)+shift_a)

x'' = x' + gate_m · MLP((1+scale_m)Norm(x')+shift_m)

时间 embedding 不只是拼接到输入，它调节每个 block 的 scale、shift 和 gate，使网络知道当前处于哪一个采样时间。

代码：text fusion 与联合序列

# mmdit.py
context = self.txtfusion(context, mask=txtmask)
context = self.txtmlp(context)
combined = torch.cat((context, img), dim=1)
...
for block in self.blocks:
    combined = block(combined, tvec, freqs, mask)
final = self.last(combined, t)
output = final[:, txtlen : txtlen + imglen, :]

注意最后只取 image 区间，因为 text 区间是条件上下文，不是要 decode 的 latent velocity。forward 里每个切片都对应一个明确的语义。

实验：tiny MMDiT 的 shape 和 finite check

cfg = SingleMMDiTConfig(
    features=128, tdim=32, txtdim=64,
    heads=4, kvheads=2, multiplier=2,
    layers=1, patch=2, channels=4,
    txtlayers=3, txtheads=4, txtkvheads=4,
)
out = model(img, context, t, pos, mask)
assert out.shape == (1, image_len, 16)
assert torch.isfinite(out).all()

tiny 实验用于验证接口：它检查 QKV、GQA、RoPE、text fusion、padding 和输出切片能在小配置上连通。质量结论需要真实 checkpoint、正式协议和测量结果。

Evidence and scope

Source：text fusion/block、SingleStreamDiT；官方 Krea2 transformer config。

Teaching use：用一条 forward 解释多模态 token 的联合建模。

Scope：当前仓库只实现推理主干；训练策略、数据配比和蒸馏过程不由这些类直接证明。

Retrieval practice

  1. 为什么 text 和 image 的 feature dimension 在进入 combined 前必须相同？
  2. 为什么最终只取 image token 输出？
  3. 时间调制放在每个 block 内会提供哪些额外表达能力？
分层答案

标准答案：combined 是沿 sequence 维拼接，所以最后 feature 维必须一致。只有 image token 对应 latent 的 velocity；text token 是条件。block 内调制让每层都能根据 t 改变归一化后的表示、残差门控和 MLP 行为，表达更丰富的时间依赖。

常见误解：MMDiT 的“multi-modal”表示条件先经过专门融合，再进入联合流；各模态在同一主干中发生信息交换。
