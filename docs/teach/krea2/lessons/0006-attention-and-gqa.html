Lesson 06 / Mechanism

Attention、multi-head attention 与 GQA

Attention 由可计算的相似度、归一化和加权求和构成；“模型注意力集中”只是缺少数学定义的直觉描述。

前置：token + shape产物：attention trace实验：CPU 小矩阵

Remember now

Q 决定“我在找什么”，K 决定“我有什么标签”，V 决定“被取走的内容”。attention 输出的是 V 的加权组合，不是把 token 直接复制。

直觉：查询、标签、内容

假设一个 token 想知道句子里的“颜色”信息：它产生 query；每个 token 产生 key 表示“我可能包含什么”；对应的 value 携带真正内容。query 与 key 越相似，value 的权重越大。

Q: current question K: searchable labels V: content | | | +-------- similarity ------+ | softmax weights --------------+ weighted values -> output

公式：从点积到输出

对单个 head：

Attention(Q,K,V) = softmax(QK^T / sqrt(d_k) + M)V

M 是 mask；无效位置可以被设成极小值，等价于 softmax 后权重为零。multi-head 将 hidden dimension 切成多个子空间并行计算，再拼回去。

Krea 2 的 MMDiT 使用 heads=48、kvheads=12，query 有 48 个 head，而 key/value 只有 12 个。每个 KV head 被多个 query head 共享，这就是 grouped-query attention。

代码：QKV、head reshape 和 SDPA

# mmdit.py
q, k, v = self.wq(qkv), self.wk(qkv), self.wv(qkv)
q = rearrange(q, "B L (H D) -> B H L D", H=self.heads)
k = rearrange(k, "B L (H D) -> B H L D", H=self.kvheads)
v = rearrange(v, "B L (H D) -> B H L D", H=self.kvheads)
q, k, v = self.qknorm(q, k, v)
out = F.scaled_dot_product_attention(q, k, v,
                                     attn_mask=mask,
                                     enable_gqa=self.gqa)

关键 shape 是 Q=(B,48,L,128)、K/V=(B,12,L,128)。真正的 backend 在 mmdit.py:40-44 被固定为 CUDNN attention，这解释了后面硬件实验的重要性。

实验：手工验证 mask 和 GQA

q = torch.randn(1, 4, 3, 8)
k = torch.randn(1, 2, 3, 8)
v = torch.randn(1, 2, 3, 8)
mask = torch.tensor([[[[True, True, False],
                       [True, True, False],
                       [True, True, False]]]])
out = torch.nn.functional.scaled_dot_product_attention(
    q, k, v, attn_mask=mask, enable_gqa=True
)
assert out.shape == q.shape

再把第三列 mask 改成 True，比较输出变化；再把所有 mask 设为 False，观察 backend 和数值行为。实验的教学目标是把“mask 屏蔽未来/无效 token”变成可观察差异。

Evidence and scope

Source：mmdit.py attention/_mask；Attention 公式来自 Transformer 原始论文。

Teaching use：把抽象 Q/K/V 映射到真实 GQA tensor contract。

Scope：公式解释 attention 机制；实际速度和可用 backend 必须以目标 Torch、GPU 和 mask 形状实验为准。

Retrieval practice

  1. 为什么 Q 的 head 数可以和 K/V 不同？
  2. mask 在 softmax 前后分别意味着什么？
  3. 为什么一个 attention 公式在 CPU 上能算，不代表本项目能在 CPU 上跑？
分层答案

标准答案：GQA 让多个 query head 共享较少的 K/V head，减少 KV 参数和缓存。mask 在 logits 上加限制，softmax 后无效位置的权重接近零。项目显式选择 CUDNN backend，且还依赖 CUDA/Triton/compile，所以数学可计算性与工程 backend 可用性是两件事。

常见误解：GQA 不是把 query 数量删掉；它只改变 K/V 的共享方式。
