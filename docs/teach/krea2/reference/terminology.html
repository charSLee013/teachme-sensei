Reference / Shared Vocabulary

Terminology and Scope

统一术语，才能把中文直觉、英文论文、代码变量和实验表格指向同一个对象。

先记住

当一个词在课程中有特殊定义时，以“当前代码中的 operational meaning”为准，并在论文里给出符号或实现位置。漂亮但模糊的术语不能替代定义。

核心词表

术语本课程的工作定义代码/证据锚点常见混淆
checkpoint某一组固定模型权重和配置的可加载状态。inference.py:load、官方 HF model card不是“模型架构”本身，也不是一次运行的配置。
latentVAE 编解码器内部的连续表示；Krea 2 采样在 latent space 中进行。autoencoder.py与 image token 含义不同；image token 描述 patchify 后的布局。
image tokenlatent 的局部 patch 被排列成的 sequence element。sampling.py:prepare与 tokenizer 产生的文本 token 含义不同。
condition/context由文本编码器提供、供 MMDiT 使用的 hidden states。encoder.py:forward不是 prompt 原字符串，也不是 attention mask。
velocityflow-matching 路径上模型预测的方向/速度，Euler 用它更新 latent。sampling.py:model output不是最终图像，也不是训练 loss 的名字。
guidance / CFG用 conditional 与 unconditional 预测的差值进行条件外推。sampling.py:CFG不同库的 scale 参数可能采用不同公式。
timestep schedule采样过程实际访问的时间点列表；它由 steps、mu、sigma 和分辨率关系共同决定。sampling.py:timesteps不只是 steps 数。
Raw / Turbo官方公开的两个 Krea 2 checkpoint 定位：Raw 为基础模型，Turbo 为少步蒸馏模型。README不是同一个 checkpoint 的两个 UI 模式。
evidence能支持一个明确 claim 的代码、公式、原始输出、统计或官方文档。第 17 课“我觉得图更好看”只能是观察，不自动是 evidence。
scopeclaim 成立的条件边界：checkpoint、分辨率、prompt、seed、硬件、指标和实验区间。第 24 课不是论文最后随便加的一句免责声明。

CCF、ACM MM 与本课程边界

本课程把“CCF 顶级编著者”解释为：能读原始论文、提出可检验问题、实现可复查实验、写出有证据和边界的英文研究论文，并能回应审稿意见。它不把某一次会议录用、某个排名或某个指标当成学习结果。

ACM MM 的具体模板、匿名规则、页数和截止时间会按当期官方 CFP 变化；论文写作时应核对当期官方页面。CCF 推荐目录也可能更新，不能把课程生成的静态页面当作当前目录的权威副本。

Evidence and scope

Source：ACM proceedings template、ACM Multimedia conference site、CCF 学术评价分类目录页面、本地 README。

Teaching use：减少“同词不同义”，让代码变量、图表和英文句子互相可追踪。

Scope：网页可能更新；写作与投稿前记录访问日期、页面标题和适用年份。

用法：开始写论文时，把本页词表中的词全部替换成定义过的对象，不用“质量、效果、理解、优化”这类无指标名词单独承担结论。
