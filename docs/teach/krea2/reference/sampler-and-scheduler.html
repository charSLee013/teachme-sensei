Sampler and Scheduler Reference

记录 timestep schedule、Euler 更新、CFG 和分辨率对齐时，使用同一套定义，避免把不同库的参数名当成同一个变量。

先记住

sampler 决定如何沿着模型预测的速度场走；scheduler 决定在哪些时间点走。两者都改变输出，不能只报告“用了 28 steps”而不报告时间网格。

Resolution-aware timestep shift

当前实现先构造从 1 到 0 的均匀网格，再将每个时间点变换：

s(t;mu,sigma) = exp(mu) / (exp(mu) + (1/t - 1)^sigma)

当 mu 未显式传入时，先令 S=N 为 image sequence length，按照两端点线性插值：

mu(S) = y1 + (S-x1) * (y2-y1)/(x2-x1)

这里的 x1=(minres/16)^2、x2=(maxres/16)^2 只是当前代码定义的分辨率参考点；改变 patch、compression 或 min/max resolution 就会改变解释。不要把 mu 误读为“模型权重里的一个参数”。

ts = torch.linspace(1, 0, steps + 1)
if mu is None:
    slope = (y2 - y1) / (x2 - x1)
    mu = slope * seq_len + (y1 - slope * x1)
ts = math.exp(mu) / (math.exp(mu) + (1.0 / ts - 1.0) ** sigma)

Euler + CFG 的逐步语义

t0=1 ------------------------------> tK=0 dt = t_next - t_current (negative) v_cond = model(x_t, context, t) v_uncond = model(x_t, empty_context, t) [only if cfg > 0] v = v_cond + g * (v_cond - v_uncond) x_next = x_t + dt * v
for tcurrent, tnext in zip(ts[:-1], ts[1:]):
    t = torch.full((batch,), tcurrent, device=device, dtype=x.dtype)
    cond = model(img=x, context=txt, t=t, pos=pos, mask=mask)
    if guidance > 0:
        uncond = model(img=x, context=untxt, t=t, pos=unpos, mask=unmask)
        velocity = cond + guidance * (cond - uncond)
    else:
        velocity = cond
    x = x + (tnext - tcurrent) * velocity

注意 dt 为负，因为当前代码从 1 积分到 0。若把符号改成正数，模型仍然可能运行，但轨迹方向已经被改变，不能把它称为同一个 sampler。

公平消融 protocol

  1. 只改变一个因素：steps、guidance、mu 或 resolution；其余参数写入配置快照。
  2. 同一 prompt 集合使用同一 seed 集合；批次内第 i 张使用 seed+i 的规则也要固定。
  3. 预热编译和 kernel 后再计时；分别记录 encode、denoise、decode 和端到端耗时。
  4. 报告 peak allocated/reserved memory，并说明是否包含模型加载和编译缓存。
  5. 质量、文本对齐、速度、显存一起看；某项变好不能自动代表总效用变好。
  6. 保存完整 timestep list 或可重建它的所有参数，不只保存 steps 数。

边界提醒

不同框架可能把 CFG scale 定义为 v_u + s(v_c-v_u)，而当前代码使用 v_c + g(v_c-v_u)。这两套参数相差一个平移关系。跨实现比较时必须写出公式，不能只把数字放在同一列。
