Reference / Reproducibility

Experiment Record Template

每次运行都留下足够的身份、配置、环境、原始结果和解释记录。没有记录的成功不能成为论文证据。

先记住

实验记录不是运行结束后凭记忆补写的总结；它是运行前冻结变量、运行中采集测量、运行后保留原始输出的单一事实来源。

Run identity

run_id: 2026-07-13-k2-raw-s000-g035
question: Does CFG improve text alignment at fixed seed and steps?
checkpoint: krea/Krea-2-Raw
checkpoint_file_sha256: <fill before writing claims>
code_commit: <git commit or working-tree diff id>
status: planned | completed | failed | excluded
owner: <name>
created_at_utc: <YYYY-MM-DD HH:MM:SS>

run_id 要能从文件名、表格行和图表 caption 反查到唯一记录。失败运行也分配 id，不要让失败样本从历史中消失。

Interpretation log

  1. Observation：只写测量到的事实，例如“CFG=3.5 的 alignment median 高于 CFG=0”。
  2. Candidate explanation：写可检验的机制，例如“条件外推增强了 prompt-specific velocity”。
  3. Alternative explanation：列出 prompt composition、seed variance、metric bias、kernel warm-up 等竞争解释。
  4. Control needed：写下一次只改变一个变量的运行。
  5. Claim boundary：说明当前证据支持到哪，不支持哪。
observation: <fact>
candidate_explanation: <mechanism>
alternative_explanations: [<...>]
control_run_ids: [<...>]
claim_boundary: <conditioned conclusion>

失败处理

OOM、编译失败、坏图、NaN、超时和指标脚本报错都保留 stdout/stderr、配置和 run id。只有在预先定义的排除规则满足时才从聚合统计中排除，并在 appendix 或 limitation 中计数。

Evidence and scope

Source：第 15 课、第 16 课、第 17 课、inference.py CLI。

Teaching use：把一次运行变成可追踪记录，再把多次运行汇总成论文表格。

Scope：模板不替学习者选择正确指标；指标必须和研究问题、图像任务及可用资源匹配。

用法：先复制模板并填“运行前”字段，再运行 sampler；不要在看到结果之后才决定比较条件。
