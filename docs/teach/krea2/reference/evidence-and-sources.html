ACM Multimedia and CCF Context

来源

  • ACM proceedings templates: ACM Publication Workflow and Templates
  • ACM Multimedia conference site: ACM Multimedia
  • CCF category page: CCF 学术评价分类目录
  • Course writing checklist: paper-writing-checklist.html

支持内容

The ACM sources support the course's practice of checking the current official template, publication workflow, author instructions, and venue-specific call for papers rather than relying on a static tutorial. The CCF page is included only as a pointer to the current official classification information and as a reminder that categories and versions must be verified at the time of use.

教学用途

The course defines “CCF top-level author” as a research capability target: precise problem framing, primary-source reading, implementable method description, controlled experiments, honest limitations, and evidence-based rebuttal. ACM MM style is used as the writing target, while CCF status is treated as external context, not as a promise of acceptance.

适用范围

Conference dates, page limits, anonymous-review rules, formatting, reproducibility requirements, and CCF classifications can change. The reader must record the page title, applicable year, access date, and any official revision before submission. This note is not legal, institutional, or acceptance advice.

Diffusers Krea 2 Pipeline

来源

  • Official Diffusers pipeline source: pipeline_krea2.py
  • Diffusers Krea 2 pipeline directory: src/diffusers/pipelines/krea2
  • Krea model index: model_index.json
  • Local functional sampler: sampling.py

支持内容

The official Diffusers implementation is a useful independent implementation to compare against the repository's minimal functional sampler. It helps identify which objects are stable conceptual interfaces (scheduler, transformer, VAE, text encoder) and which details are library-specific (argument names, CFG convention, device movement, output packaging). The model index also shows how a published pipeline records component relationships.

教学用途

Students perform a two-column trace: local sample versus the official Diffusers pipeline. For each shared step they write the input/output shape and then list any semantic difference. This trains implementation literacy without assuming that two APIs with the same word “guidance” have the same equation.

适用范围

The two implementations are not automatically numerically interchangeable. Different revisions, schedulers, weight loading, padding, precision, and postprocessing can change outputs. Use the official source as a comparison and documentation source; use the exact local code plus checkpoint hash as the source of truth for the capstone's measured protocol.

Flow Matching and Guidance

来源

  • Flow matching: Flow Matching for Generative Modeling
  • Classifier-free guidance: Classifier-Free Diffusion Guidance
  • Local schedule and Euler update: sampling.py
  • Public Krea sampler examples: README.md

支持内容

Flow Matching supplies the vector-field view: a model learns a velocity that can be integrated along a chosen path. Classifier-free guidance supplies the conditional/unconditional extrapolation idea. The local sampler makes these operational through a resolution-aware time shift, explicit Euler updates, and a CFG branch that evaluates conditional and unconditional contexts separately.

教学用途

The four-layer lesson is deliberately sequential: first imagine a particle following a field, then write x_next = x + dt * v, then inspect the exact loop and CFG formula, then run a two-dimensional toy field before spending GPU time. The capstone treats steps, mu, sigma, guidance, and resolution as experimental variables rather than decorative CLI flags.

适用范围

A flow-matching paper does not determine a particular Krea 2 schedule, checkpoint distillation recipe, or best hyperparameter. The time-shift equation in the course is a local code fact. The meaning of a Turbo recommendation is checkpoint-specific and must not be generalized without a controlled experiment.

Krea 2 Official Sources

来源

  • Krea open repository: krea-ai/krea-2
  • Official model card, Raw: krea/Krea-2-Raw
  • Official model card, Turbo: krea/Krea-2-Turbo
  • Official technical report page: Krea 2 technical report
  • Official model index: Krea-2-Raw model_index.json
  • Local license and safety documents: docs/

支持内容

The official README supports the public distinction used in the course: Raw is described as an undistilled base checkpoint, while Turbo is described as an 8-step distilled checkpoint. It also supplies the repository's example starting settings, the OSS_RAW/OSS_TURBO loading convention, and the recommendation to train LoRA on Raw and apply it to Turbo. The official model cards and repository are the first places to check for checkpoint-specific updates.

教学用途

Students first separate an official statement from a measured result. For example, “Turbo is presented as an 8-step distilled checkpoint” is source-backed context; “CFG=0 is best for our prompt set” is a new experimental claim requiring records. The source also anchors the license and safety lesson: downloading weights does not remove deployment obligations.

适用范围

Official documentation can change. Record the access date and the exact checkpoint revision or file hash for a paper experiment. Marketing claims, leaderboard positions, and hosted-product behavior remain contextual claims outside the controlled evidence set for the local inference ablation. Legal interpretation belongs to the license text and qualified counsel, not to this course summary.

Local Krea 2 Codebase

来源

  • Repository root: README.md
  • CLI and module assembly: inference.py
  • Text conditioning: encoder.py
  • Autoencoder wrapper: autoencoder.py
  • Sampler and tensor preparation: sampling.py
  • DiT/MMDiT implementation: mmdit.py
  • Dependency contract: pyproject.toml

支持内容

This source set supports the course's claims about the open inference path in this working tree: Qwen3-VL conditions text, a Qwen Image VAE provides a 16-channel f8 latent, sampling.py patchifies the latent and integrates a flow-matching trajectory, and mmdit.py predicts image-token velocities. The exact default values, padding behavior, CFG branch, and checkpoint selection are read from the current files rather than inferred from a diagram.

教学用途

The lessons use local line anchors to force a claim back to executable code. A learner should be able to start at the CLI, follow _pipeline, inspect the tensor contract at sample, and then inspect the model's text/image split. The reference pages are deliberately lookup-oriented; the lessons provide the explanation and reconstruction exercises.

适用范围

This evidence describes inference code, not the private training recipe, training data, internal production serving stack, or the full behavior of an unprovided checkpoint. The repository currently contains no downloaded Raw/Turbo weights, so the course must distinguish code-path tests from full real-checkpoint image results. A dirty working tree or a changed dependency version invalidates claims that depend on exact runtime behavior.

Qwen3-VL and Qwen Image Components

来源

  • Text model card: Qwen/Qwen3-VL-4B-Instruct
  • Qwen VL repository: QwenLM/Qwen3-VL
  • Qwen Image model card: Qwen/Qwen-Image
  • Qwen Image repository: QwenLM/Qwen-Image
  • Local conditioning wrapper: encoder.py
  • Local VAE wrapper: autoencoder.py
  • Diffusers Qwen Image VAE implementation: AutoencoderKLQwenImage

支持内容

The local project names Qwen/Qwen3-VL-4B-Instruct as its text model and loads the Qwen Image VAE from the vae subfolder of Qwen/Qwen-Image. The local wrapper selects hidden states from layers (2, 5, ..., 35), removes a fixed prompt-template prefix, truncates to 512 conditioning positions, and exposes the Qwen VAE's f8/16-channel latent contract. Those exact wrapper choices are local implementation facts; the Qwen documentation supports understanding the component families and public model interfaces.

教学用途

The learner compares “what Qwen provides” with “what Krea's wrapper asks for.” This is the bridge from a model card to an integration contract: tokenizer behavior, hidden-state layout, dtype/device movement, latent normalization, and decode shape all need to agree. The course's toy tests can verify layouts without claiming that a tiny test reproduces the full checkpoint.

适用范围

Component documentation does not prove that Krea used an unchanged component during training, nor does it reveal a private prompt preprocessing recipe. For a real run, pin the exact model revision and Transformers/Diffusers versions, save the wrapper configuration, and record whether remote code or cached files were involved.

Reproducibility and Evaluation

来源

  • ACM artifact review and badging: ACM Artifact Review and Badging
  • NeurIPS paper checklist: Paper Checklist
  • CLIP: Learning Transferable Visual Models From Natural Language Supervision
  • FID: GANs Trained by a Two Time-Scale Update Rule Converge to a Local Nash Equilibrium
  • LPIPS: The Unreasonable Effectiveness of Deep Features as a Perceptual Metric
  • Course experiment record: experiment-record-template.html

支持内容

The artifact and checklist sources support the discipline of reporting enough implementation detail for another researcher to inspect and reproduce a result. CLIP, FID, and LPIPS support discussion of common image/text or perceptual metrics while also motivating metric-specific caution: a metric is an operational measurement, not a complete definition of “quality.”

教学用途

The course teaches a metric triangulation: pair a named metric with prompt-level or human-readable error analysis, keep seeds and prompt sets fixed for comparisons, report latency and memory separately, and preserve failures. The learner must be able to regenerate a table from raw records and explain why a metric might disagree with visual inspection.

适用范围

No single metric establishes overall image quality, alignment, safety, or usefulness. The cited checklists are guidance, not a guarantee of reproducibility; exact hardware, software, model revision, data, and nondeterministic kernels still matter. The course does not prescribe a universal metric suite for every image-generation task.

Transformer, Attention, RoPE, GQA, and DiT Papers

来源

  • Transformer: Attention Is All You Need
  • RoPE: RoFormer: Enhanced Transformer with Rotary Position Embedding
  • GQA: GQA: Training Generalized Multi-Query Transformer Models from Multi-Head Checkpoints
  • DiT: Scalable Diffusion Models with Transformers
  • Local attention and positional code: mmdit.py

支持内容

The Transformer paper supplies the query/key/value attention vocabulary and residual-stack mental model. RoPE supports the positional-rotation intuition used when reading the local three-axis positional encoding. GQA supports the distinction between query heads and fewer key/value heads. DiT supports the design argument for using a transformer as the denoising backbone and for relating token count to compute.

教学用途

The lessons treat these papers as conceptual ancestors, not as proof that Krea 2 is a literal reproduction. The learner maps each paper's abstraction to the local code: where Q/K/V are formed, where heads are grouped, where positions become frequencies, where text and image streams are fused, and where the final image-token velocity is projected.

适用范围

The papers describe their own architectures and experiments. They do not establish Krea 2's private training choices or guarantee that a local implementation has identical complexity. Claims about Krea's exact layer counts, dimensions, and head counts come from local config and official public artifacts, and claims about performance require measurement on the selected hardware.

VAE and Diffusion Foundations

来源

  • VAE: Auto-Encoding Variational Bayes
  • DDPM: Denoising Diffusion Probabilistic Models
  • Latent diffusion: High-Resolution Image Synthesis with Latent Diffusion Models
  • Local VAE wrapper: autoencoder.py
  • Local sampling path: sampling.py

支持内容

The VAE paper supports the encoder/latent/decoder vocabulary and the idea that a learned latent representation can mediate generation. DDPM supports the noise-prediction diffusion family and the reason a model is evaluated across a sequence of noise levels. Latent Diffusion supports moving the generative process into a compressed representation to reduce spatial cost. These sources provide foundation, not a claim that the local Krea 2 sampler is a DDPM implementation.

教学用途

Students explicitly compare discrete diffusion language with the local flow-matching language: what is the state, what is the model output, what is the time parameter, and how is the state updated? This prevents “all noise-based image models are the same” from becoming an unexamined shortcut.

适用范围

The papers' notation and objectives differ from Krea 2's public implementation. The course uses them to build intuition and distinguish families, then returns to sampling.py for the operational equation actually measured. The Qwen VAE's normalization constants and decoder behavior must be read from the pinned component revision at runtime.
