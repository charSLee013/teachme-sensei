元素总数的 sanity checks

VAE latent 到 image token 的重排不能改变元素总数：

16 * (H/8) * (W/8) = ((H/16)*(W/16)) * (16*2*2)

例如 1024x1024 时，latent 是 (B,16,128,128)，image token 是 (B,4096,64)。如果 token 数、patch size 或 channel 数任何一个不匹配，错误通常应该在 round-trip 检查处暴露。

import torch
from einops import rearrange

x = torch.randn(2, 16, 128, 128)
tokens = rearrange(x, "b c (h ph) (w pw) -> b (h w) (c ph pw)", ph=2, pw=2)
y = rearrange(tokens, "b (h w) (c ph pw) -> b c (h ph) (w pw)",
              h=64, w=64, c=16, ph=2, pw=2)
assert tokens.shape == (2, 4096, 64)
assert torch.equal(x, y)

对齐规则

pixel width/height divisible by ae.compression=8 and by patch=2 after latent conversion => original width/height divisible by 16 1024 -> latent 128 -> token grid 64 -> N=4096 1030 -> roundup to 1040 -> latent 130 -> token grid 65 -> N=4225

当前 sampler 会把不满足 16 倍数的输入向上补齐；这意味着“请求尺寸”和“实际计算尺寸”可能不同。实验日志必须同时记录二者，输出裁剪策略也要写清楚。

错误定位表

症状优先检查不要先做的事
mat1 and mat2 shapes cannot be multipliedpatchify 最后一维是否等于 channels*patch**2，以及 Linear 配置。不要先增大显存或改 batch。
attention mask shape errortext mask 是否 bool；combined mask 是否与 text+image 长度一致。不要用全 1 mask 掩盖 padding。
坏图但代码不报错unpatchify 的 h/w、通道顺序、VAE decode dtype 与范围。不要只检查最终 PIL 能否保存。
位置编码不匹配pos.shape[-1]==3，且 image grid 的行列坐标与 token 顺序一致。不要把 position 直接 reshape 成 hidden state。
