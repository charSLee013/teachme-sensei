Krea 2 / Multimodal Generation / Research Writing

从读懂推理代码，到写出可复现的生成模型论文

本课程把 Krea 2 作为可拆解、可测量、可质疑、可重建的研究对象，带你从基础心智模型走到 ACM MM 风格的完整论文草稿。

先记住

熟悉概念只是起点。研究能力要求把主张连接到公式、代码、实验和证据，并诚实说明它成立的条件。

唯一毕业主线

在不改变 Krea 2 checkpoint 的前提下，固定 prompt、seed、硬件和环境，系统测量分辨率感知的 flow-matching 时间调度、CFG、Raw/Turbo 设置对图像质量、文本对齐、推理速度和显存的影响，最后写成可复现 inference protocol。

prompt | v Qwen3-VL hidden states + mask ----+ | noise -> patchify -> positions ----> Krea 2 MMDiT -> Euler flow path -> VAE -> image | quality / alignment / time / memory | baseline -> ablation -> claim -> paper

课程契约

项目本课程的决定
学习者会基础 Python，缺少 ML、生成模型和科研写作体系。
语言中文讲解；英文摘要、方法、图表说明和 rebuttal 逐步训练。
路径直觉 -> 公式 -> 代码 -> 实验；每个核心机制都完整走完四层。
实验CPU toy tests 用于基础验证；真实 checkpoint 实验需要另行记录可用 GPU、显存、软件环境和测量结果。
结果一篇完整研究论文草稿、一套实验记录和一次模拟审稿答辩。
边界训练的是可验证的 CCF 高水平研究写作能力，不承诺论文录用。

学习路线

按依赖顺序阅读。每课结束后，先完成 retrieval practice，再执行 next action；不要只向下滚动看答案。

  1. 如何把 GitHub 仓库读成研究对象建立角色、对象、接口、证据和研究问题。
  2. PyTorch 张量与 shape 思维先学会看懂每个维度，再看模型。
  3. 概率、噪声与可复现随机性理解生成的随机起点。
  4. 神经网络从训练到推理明确参数、状态、梯度和推理的区别。
  5. Token、embedding 与条件输入从文字走到 hidden states。
  6. Attention、multi-head attention 与 GQA理解信息如何混合。
  7. RoPE 与三轴位置理解 token 为什么知道自己在哪。
  8. VAE 与 latent space理解图像为何先被压缩。
  9. Diffusion、噪声路径与 flow matching建立连续生成的过程模型。
  10. CFG、Euler 和采样步数把模型输出变成一条可控轨迹。
  11. DiT 与 MMDiT看懂 Krea 2 的主干网络。
  12. Krea 2 完整 pipeline trace把四个模块接成一条数据流。
  13. 从代码到张量状态逐行追踪 encoder、sampler 与 model。
  14. 编译、注意力 kernel、显存与性能理解从“能运行”走到“可部署”还需要哪些条件。
  15. 研究问题、变量与实验矩阵把好奇心变成可检验问题。
  16. 指标、统计与错误分析避免只挑好看的图片。
  17. 可复现 inference protocol固定环境、记录失败、保存证据。
  18. 如何读生成模型原始论文从摘要读到 claim-evidence。
  19. Related work 与研究空白避免把背景堆成论文。
  20. 英文摘要与 Introduction让读者知道问题为何值得解决。
  21. Method 与公式写作把实现写成别人可以复核的模型。
  22. Experiments、表格与统计叙事让数字回答研究问题。
  23. 图、caption 与视觉证据让图表本身承担论证。
  24. 限制、负结果与诚实结论把边界写成可信度。
  25. 模拟审稿人与 objection map提前寻找会被质疑的地方。
  26. Rebuttal 与答辩用证据回应质疑。
  27. 毕业论文包把实验记录整合成一篇研究论文。
  28. 最终综合与迁移从 Krea 2 迁移到新的生成系统。
