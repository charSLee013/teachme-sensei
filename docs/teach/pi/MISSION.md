## Mission

Build a repo-course teaching workspace that lets an engineer understand how Pi Agent Harness is structured, where each major mechanism lives, and what the current repository facts do and do not prove.

## Learner Outcome

After finishing the course, a learner should be able to:

- name the five workspace packages and their dependency chain;
- trace one prompt through CLI, session, agent loop, provider, and tool execution;
- distinguish stable repository facts from teaching reconstruction;
- contribute a small change without inventing a pnpm/Turbo workflow that the repo does not use.

## Delivery Boundary

This workspace teaches the current repository at `/private/tmp/pi`. It does not promise future release behavior, upstream governance beyond the checked docs, or production guidance for the experimental orchestrator.
