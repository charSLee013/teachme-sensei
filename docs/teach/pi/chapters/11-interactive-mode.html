Chapter 11: 交互模式 — TUI 实现详解

Pi 的 交互模式 是一个全功能 TUI 应用，基于 pi-tui 渲染引擎构建。本章解析 InteractiveMode 类的组件架构、流渲染、按键绑定和主题系统。

Main Line

交互模式是 Pi 把底层 agent、tool、session 事件翻译成可操作终端体验的那一层，所以它既是 UI，又是行为编排器。

Remember Now

InteractiveMode 不替代 `AgentSession`。它订阅事件、渲染状态、处理按键，然后把真正的业务动作交回 runtime。

11.1 InteractiveMode 概览

InteractiveMode 是 TUI 的主控制器，负责渲染和用户交互，业务逻辑委托给 AgentSession。它订阅 agent 事件流并实时更新 UI。

export class InteractiveMode {
    private runtimeHost: AgentSessionRuntime;
    private ui: TUI;
    private chatContainer: Container;
    private editor: EditorComponent;
    private footer: FooterComponent;
    private keybindings: KeybindingsManager;
    private themeController: InteractiveThemeController;

    // 流式消息追踪
    private streamingComponent: AssistantMessageComponent | undefined;
    private streamingMessage: AssistantMessage | undefined;
    private pendingTools = new Map<string, ToolExecutionComponent>();

    async run(): Promise<void> {
        await this.init();
        // 主交互循环
        while (true) {
            const userInput = await this.getUserInput();
            await this.session.prompt(userInput);
        }
    }
}

11.2 组件架构

主要组件一览

  • ArminComponent：品牌标识/吉祥物渲染
  • AssistantMessageComponent：渲染助手回复（支持 thinking 块、tool calls）
  • UserMessageComponent：渲染用户消息
  • EditorComponent / CustomEditor：输入框，支持 autocomplete、bash mode、外部编辑器
  • FooterComponent：底部状态栏
  • ToolExecutionComponent：工具执行结果展示（支持展开/折叠）
  • SessionSelectorComponent：会话选择器（搜索、过滤、重命名、删除）
  • ModelSelectorComponent：模型选择器
  • TreeSelectorComponent：会话树导航器
  • TrustSelectorComponent：项目信任选择器
  • CompactionSummaryMessageComponent / BranchSummaryMessageComponent：压缩/分支摘要渲染

11.3 流式渲染

当模型开始流式响应时，InteractiveMode 通过事件订阅实时更新 UI：

// 事件处理核心逻辑
switch (event.type) {
    case "message_start":
        if (event.message.role === "assistant") {
            this.streamingComponent = new AssistantMessageComponent(
                undefined, this.hideThinkingBlock,
                this.getMarkdownThemeWithSettings(),
                this.hiddenThinkingLabel, this.outputPad,
            );
            this.streamingMessage = event.message;
            this.chatContainer.addChild(this.streamingComponent);
            this.streamingComponent.updateContent(this.streamingMessage);
        }
        break;

    case "message_update":
        if (this.streamingComponent && event.message.role === "assistant") {
            this.streamingMessage = event.message;
            this.streamingComponent.updateContent(this.streamingMessage);
            // 检测新 toolCall，创建 ToolExecutionComponent
            for (const content of this.streamingMessage.content) {
                if (content.type === "toolCall" && !this.pendingTools.has(content.id)) {
                    const component = new ToolExecutionComponent(content.name, ...);
                    this.pendingTools.set(content.id, component);
                }
            }
        }
        break;

    case "message_end":
        if (this.streamingComponent && event.message.role === "assistant") {
            this.streamingComponent.updateContent(this.streamingMessage);
            this.streamingComponent = undefined;  // 停止流式更新
        }
        break;
}
流式 vs 一次性渲染

流式期间使用同一个 AssistantMessageComponent 实例反复调用 updateContent()，只在 message_end 后销毁。这避免了组件频繁创建销毁的开销。

模式特定按键

不同模式下按键行为不同：

  • 普通模式：Enter 发送消息，! 前缀进入 bash 模式
  • 选择器模式：j/k 导航，Enter 选择，q/Esc 退出
  • 树选择器模式：Ctrl+K/Ctrl+J 折叠/展开，e 编辑消息
  • 外部编辑器模式：在 $EDITOR 中编辑，保存退出后发送

11.5 自动补全

编辑器支持多种补全源，通过 CombinedAutocompleteProvider 组合：

private createBaseAutocompleteProvider(): AutocompleteProvider {
    // 1. 内置斜杠命令
    const slashCommands: SlashCommand[] = BUILTIN_SLASH_COMMANDS.map(cmd => ({
        name: cmd.name, description: cmd.description,
    }));

    // 2. 模型选择器的模糊匹配
    const modelCommand = slashCommands.find(c => c.name === "model");
    modelCommand.getArgumentCompletions = (prefix) => {
        const items = models.map(m => ({ label: `${m.provider}/${m.id}`, ... }));
        return fuzzyFilter(items, prefix, getModelSearchText);
    };

    // 3. 提示模板
    const templateCommands = this.session.promptTemplates.map(cmd => ({
        name: cmd.name, description: cmd.description,
    }));

    // 4. 扩展注册的命令
    const extensionCommands = this.session.extensionRunner
        .getRegisteredCommands()
        .filter(cmd => !builtinCommandNames.has(cmd.name));

    // 5. Skill 命令 (skill:name)
    const skillCommandList = this.session.resourceLoader.getSkills().skills.map(skill => ({
        name: `skill:${skill.name}`, description: skill.description,
    }));

    return new CombinedAutocompleteProvider(
        [...slashCommands, ...templateCommands, ...extensionCommands, ...skillCommandList],
        this.sessionManager.getCwd(), this.fdPath,
    );
}

文件路径补全

输入 @ 触发文件路径补全，使用系统 fd 命令搜索文件，支持模糊匹配。

11.10 Quiz

1. InteractiveMode 如何接收模型的流式响应？
  • 通过轮询 session.getMessages()
  • 通过订阅 agent 事件流（message_start / message_update / message_end）
  • 通过 WebSocket 连接
  • 通过共享内存 buffer
InteractiveMode 订阅 AgentSession 的事件流，根据事件类型更新 UI 组件。这是响应式架构的核心。
2. CombinedAutocompleteProvider 不处理以下哪种补全？
  • 内置斜杠命令（如 /model）
  • 扩展注册的命令
  • 代码语义补全（如 LSP 补全）
  • Skill 命令（skill:name）
Pi 的 autocomplete 处理命令、模型、模板、扩展和 skill 的补全。代码语义补全由 LSP 或其他专用工具处理，不在 autocomplete provider 范围内。
3. 主题系统的颜色模式 ansi 和 truecolor 的区别是？
  • ansi 支持更多颜色，truecolor 只支持 256 色
  • ansi 使用 256 色或 16 色，truecolor 使用 24 位 RGB
  • 没有区别，只是名字不同
  • truecolor 仅适用于 light 主题
truecolor（24 位）支持约 1677 万色，但需要终端支持。ansi 模式兼容性更好，适用于老旧终端。
4. 项目信任流的主要目的是什么？
  • 验证用户身份
  • 确认用户是否允许 Pi 加载项目本地资源（skills、extensions 等）
  • 加密会话文件
  • 限制 API 调用频率
项目信任流防止恶意仓库通过 Pi 的 skills/extensions 机制执行未授权代码。用户必须显式信任项目后，Pi 才会加载本地资源。
