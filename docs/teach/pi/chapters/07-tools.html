第 7 章

内置工具系统 — 7 个工具的设计

理解工具的定义、验证、执行模式

Main Line

工具系统的关键不是“能做多少事”，而是怎样把模型可见的 schema、运行时执行函数和文件变更纪律拆开。

Remember Now

当前内置工具是 `read`、`bash`、`edit`、`write`、`grep`、`find`、`ls`。Web 搜索和 agent delegation 不属于这组内置表面。

7.1 工具架构总览

Pi 的工具有两个核心概念：

  • ToolDefinition：描述工具的名称、描述和参数 JSON Schema，发送给 LLM 让它知道如何调用
  • AgentTool：包含 ToolDefinition + 实际执行函数

7.2 工具工厂模式

// core/tools/index.ts
export type ToolName = "read" | "bash" | "edit" | "write" | "grep" | "find" | "ls";

export function createTool(toolName: ToolName, cwd: string, options?: ToolsOptions): Tool {
  switch (toolName) {
    case "read":   return createReadTool(cwd, options?.read);
    case "bash":   return createBashTool(cwd, options?.bash);
    case "edit":   return createEditTool(cwd, options?.edit);
    case "write":  return createWriteTool(cwd, options?.write);
    case "grep":   return createGrepTool(cwd, options?.grep);
    case "find":   return createFindTool(cwd, options?.find);
    case "ls":     return createLsTool(cwd, options?.ls);
  }
}

export function createToolDefinition(toolName: ToolName, cwd: string, options?: ToolsOptions): ToolDef {
  // 同样的 switch，但返回 definition（给 LLM 的 schema）
}

// 批量创建
export function createCodingTools(cwd: string, options?: ToolsOptions): Tool[] {
  return [createReadTool(cwd), createBashTool(cwd), createEditTool(cwd), createWriteTool(cwd)];
}

export function createReadOnlyTools(cwd: string, options?: ToolsOptions): Tool[] {
  return [createReadTool(cwd), createGrepTool(cwd), createFindTool(cwd), createLsTool(cwd)];
}

7.3 逐一介绍 7 个工具

工具 用途 关键参数 实现方式
bash 执行 Shell 命令 command, timeout, description child_process.spawn
edit 文件编辑（diff 格式） file_path, old_string, new_string, replace_all 字符串匹配 + 替换
read 读取文件内容 file_path, offset, limit, file_path (images) fs.readFile + 截断
write 创建/覆盖文件 file_path, content fs.writeFile
grep 内容搜索 pattern, path, glob, output_mode, context ripgrep (rg) 封装
find 文件查找 pattern, path, type, glob glob 库
ls 目录列表 path, ignore fs.readdir + ignore 库

7.4 Edit 工具深度走读

Edit 工具是最复杂的内置工具，它使用统一 diff 格式进行精确编辑：

TypeBox Schema

export const EditToolInput = Type.Object({
  file_path: Type.String({ description: "Absolute path to the file to modify" }),
  old_string: Type.String({ description: "The exact text to replace" }),
  new_string: Type.String({ description: "The replacement text" }),
  replace_all: Type.Boolean({ description: "Replace all occurrences", default: false }),
});

发送给 LLM 的 JSON Schema

{
  "name": "edit",
  "description": "Edit a file by replacing exact text...",
  "parameters": {
    "type": "object",
    "properties": {
      "file_path": { "type": "string", "description": "Absolute path to the file" },
      "old_string": { "type": "string", "description": "The exact text to replace" },
      "new_string": { "type": "string", "description": "The replacement text" },
      "replace_all": { "type": "boolean", "default": false }
    },
    "required": ["file_path", "old_string", "new_string"]
  }
}

执行逻辑

export function createEditTool(cwd: string, options?: EditToolOptions): Tool {
  return {
    name: "edit",
    description: "Edit a file by replacing exact text...",
    parameters: EditToolInput,
    label: "Edit",
    async execute(toolCallId, params, signal) {
      const filePath = resolvePath(cwd, params.file_path);
      const content = readFileSync(filePath, "utf-8");

      // 精确匹配 old_string
      const occurrences = countOccurrences(content, params.old_string);
      if (occurrences === 0) throw new Error("old_string not found in file");
      if (occurrences > 1 && !params.replace_all) throw new Error("Multiple matches found");

      // 执行替换
      const newContent = params.replace_all
        ? content.replaceAll(params.old_string, params.new_string)
        : content.replace(params.old_string, params.new_string);

      writeFileSync(filePath, newContent);

      // 生成 diff 返回给 LLM
      const diff = generateDiffString(content, newContent);
      return {
        content: [{ type: "text", text: diff }],
        details: { filePath, occurrences },
      };
    },
  };
}

7.7 路径安全

// path-utils.ts
export function resolvePath(cwd: string, inputPath: string): string {
  // 1. 转换为绝对路径
  const resolved = path.resolve(cwd, inputPath);
  // 2. 验证路径在 cwd 内（防止路径穿越）
  if (!resolved.startsWith(cwd)) {
    throw new Error("Path traversal detected");
  }
  return resolved;
}

export function normalizePath(filePath: string): string {
  // 统一路径格式，移除冗余的 ./ 和 ../
  return path.normalize(filePath);
}

7.8 小测验

Q1: ToolDefinition 和 AgentTool 的区别是什么？

A. 没有区别，是同一个东西
B. ToolDefinition 是给 LLM 的 schema，AgentTool 包含定义 + 执行函数
C. ToolDefinition 是给 UI 的，AgentTool 是给 LLM 的
D. ToolDefinition 是旧版，AgentTool 是新版
✅ 正确答案：B。ToolDefinition（ToolDef）只包含 schema 信息，AgentTool（Tool）额外包含 execute 函数。

Q2: Edit 工具如何保证精确编辑？

A. 使用正则表达式
B. 使用精确字符串匹配（old_string 必须完全匹配）
C. 使用 AST 分析
D. 使用 diff 算法
✅ 正确答案：B。Edit 工具使用精确字符串匹配，old_string 必须在文件中完全匹配（除非 replace_all=true）。

Q3: 文件变异队列的作用是什么？

A. 限制工具的并发数量
B. 保证同一文件的读写操作顺序
C. 缓存文件内容
D. 记录文件操作日志
✅ 正确答案：B。队列确保同一文件的读写操作按顺序执行，避免竞态条件。

Q4: resolvePath() 函数如何防止路径穿越？

A. 禁止绝对路径
B. 验证解析后的路径是否在 cwd 内
C. 禁止 .. 字符
D. 使用 chroot
✅ 正确答案：B。函数解析路径后检查是否以 cwd 开头，防止 ../../etc/passwd 等攻击。

Q5: Grep 工具的底层实现是什么？

A. JavaScript 正则表达式
B. ripgrep (rg) 命令行工具
C. Git grep
D. ag (Silver Searcher)
✅ 正确答案：B。Pi 的 grep 工具封装了 ripgrep，提供高性能的内容搜索。

7.9 练习

编写一个自定义 note 工具：保存带时间戳的笔记到文件。

// 参考实现框架
export function createNoteTool(cwd: string): Tool {
  return {
    name: "note",
    description: "Save a timestamped note to a file",
    parameters: Type.Object({
      content: Type.String({ description: "The note content" }),
      file: Type.Optional(Type.String({ description: "Note file path", default: ".pi/notes.md" })),
    }),
    label: "Note",
    async execute(toolCallId, { content, file }) {
      const filePath = resolvePath(cwd, file ?? ".pi/notes.md");
      const timestamp = new Date().toISOString();
      const entry = `[${timestamp}] ${content}\n`;
      appendFileSync(filePath, entry);
      return {
        content: [{ type: "text", text: `Note saved to ${file}` }],
        details: { filePath, timestamp },
      };
    },
  };
}
