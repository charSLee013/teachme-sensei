第 5 章

pi-tui — 终端 UI 库

理解差分渲染和组件系统

Main Line

pi-tui 不是装饰层，而是 Pi 交互模式能否流畅工作的基础设施。它要在终端里承担高频流式渲染、键盘输入和图像降级这些浏览器通常代劳的事情。

Remember Now

这套 TUI 的核心抽象很小：组件渲染成字符串数组，再由差分渲染器负责高效更新屏幕。

5.1 为什么自建 TUI？

Node.js 生态已有多个 TUI 库（blessed、ink、blessed-contrib），但 Pi 选择自建，原因：

  • 差分渲染：Pi 的 TUI 每秒更新数十次（流式 token），全屏重绘会导致闪烁和性能问题
  • 终端图像：需要在终端显示截图、图表，需要精确控制 Kitty/iTerm2 协议
  • 键绑定：需要支持 Kitty 键盘协议（key release、repeat 检测）
  • 集成度：与 Agent 事件系统深度集成，自定义渲染器

5.2 核心概念：Component 接口

export interface Component {
  /** 渲染组件为字符串数组（每行一个字符串） */
  render(width: number): string[];

  /** 可选：处理键盘输入（仅当组件有焦点时） */
  handleInput?(data: string): void;

  /** 是否接收 key release 事件（Kitty 协议） */
  handleRelease?(): void;
}

这是一个极其简洁的设计。所有 UI 元素都实现这个接口：文本、编辑器、选择列表、Markdown 渲染器，甚至图像。

组件组合

export class Container implements Component {
  children: Component[] = [];
  layout: "vertical" | "horizontal" = "vertical";

  render(width: number): string[] {
    // 根据布局方向，组合子组件的渲染结果
    const childLines = this.children.map(c => c.render(width));
    return this.layout === "vertical"
      ? childLines.flat()
      : mergeHorizontal(childLines, width);
  }
}

5.3 差分渲染原理

Pi 的 TUI 使用差分渲染来高效更新终端：

实现原理：TUI 维护上一帧的渲染结果。每帧渲染时，逐行比较新旧内容，只对变化的行发送 ANSI escape code 定位光标 + 覆盖写入。

5.8 小测验

Q1: 差分渲染相比全屏重绘的主要优势是？

A. 更简单的代码
B. 避免闪烁、提高性能
C. 支持更多颜色
D. 更好的跨平台兼容性
✅ 正确答案：B。差分渲染只更新变化的行，避免了全屏清除-重绘的闪烁问题，同时减少了 I/O 操作。

Q2: Component 接口的核心方法是什么？

A. mount() / unmount()
B. render(width) → string[]
C. draw(ctx)
D. update(props)
✅ 正确答案：B。所有组件实现 render(width) 方法，返回字符串数组（每行一个字符串）。

Q3: Kitty 键盘协议相比普通终端协议的优势是？

A. 支持更多颜色
B. 支持中文输入
C. 可以区分 key press 和 key release
D. 支持更长的键名
✅ 正确答案：C。Kitty 协议可以区分按键按下和松开，以及检测 key repeat。

Q4: Pi 如何在不支持图像的终端上显示图像？

A. 自动跳过图像显示
B. 使用 fallback（如文件路径或占位符）
C. 转换为 ASCII art
D. 不支持
✅ 正确答案：B。terminal-image.ts 的 imageFallback() 函数提供降级方案。
