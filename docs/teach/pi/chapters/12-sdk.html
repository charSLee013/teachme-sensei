Chapter 12: 编程式 SDK — 把 Pi 当库用

大多数用户通过 CLI 与 Pi 交互，但 Pi 本质上是一个 library。本章介绍如何通过 SDK 把 Pi 嵌入你自己的应用——自定义 UI、自动化脚本、IDE 集成，甚至测试框架。

Main Line

SDK 章节要证明的不是“Pi 也能 import”，而是 CLI 背后的关键 runtime 入口本来就可以被别的程序重用。

Remember Now

把 Pi 当库用时，最值钱的不是 shell 外壳，而是 read-only tools、runtime 事件流和 rpc/print 这类可组合入口。

12.1 Why SDK?

CLI 适合人机交互，但当你需要以下能力时，SDK 才是正解：

  • 自定义 UI：构建 Web 界面、桌面应用或移动端前端，Pi 在后台运行。
  • 自动化：在 CI/CD 流水线中批量处理代码审查、文档生成等任务。
  • IDE 集成：把 Pi 作为 LSP 后端或 VS Code 扩展的核心引擎。
  • 测试：编写集成测试时，用 SDK 驱动 agent 并断言其行为。
Info

SDK 暴露了 CLI 背后的全部能力。CLI 本身就是 SDK 的一个薄封装——理解 SDK 等于理解了整个系统的编程接口。

12.2 SDK Entry Points

SDK 的入口在 src/index.ts，它重新导出了所有公共 API：

// src/index.ts — SDK 公共入口
export { createAgentSession } from './session';
export { createAgentSessionRuntime } from './runtime';
export { createAgentSessionServices } from './services';
export { createCodingTools, createReadOnlyTools } from './tools';
export { InteractiveMode, runPrintMode, runRpcMode } from './modes';
export type { AgentSessionEvent, AgentSessionEventListener } from './agent-session';

在消费端，你只需：

import {
    createAgentSession,
    createCodingTools,
    InteractiveMode,
} from 'pi-agent';

12.3 Core API

createAgentSession()

创建并配置一个 agent 会话。这是最常用的入口：

const session = await createAgentSession({
    model: 'claude-sonnet-4-20250514',
    tools: createCodingTools(),
    systemPrompt: 'You are a helpful coding assistant.',
    workspace: '/path/to/project',
});

createAgentSessionRuntime()

创建低层运行时，暴露消息循环和 token 流：

const runtime = await createAgentSessionRuntime({
    model: 'claude-sonnet-4-20250514',
    stream: true,
    maxTurns: 10,
});

runtime.on('message', (msg) => console.log(msg.role, msg.content));
runtime.on('token', (tok) => process.stdout.write(tok));

createAgentSessionServices()

创建独立的服务实例（memory、tool registry、formatter），便于细粒度控制：

const services = await createAgentSessionServices({
    memory: { type: 'sliding-window', maxTokens: 4096 },
    tools: { registry: customRegistry },
    formatter: { provider: 'anthropic' },
});
Tip

大多数场景用 createAgentSession() 就够了。只有需要自定义消息循环或替换 formatter 时，才需要直接操作 runtime 或 services。

12.4 Tool Factories

createCodingTools()

返回完整的编码工具集——文件读写、shell 执行、搜索等：

import { createCodingTools } from 'pi-agent';

const tools = createCodingTools({
    // 可选：限制可访问的目录
    allowedPaths: ['/workspace/project'],
    // 可选：禁用危险工具
    disabledTools: ['shell_exec'],
});

console.log(tools.list().map(t => t.name));
// ['read_file', 'write_file', 'list_dir', 'search', ...]

createReadOnlyTools()

只读工具集——适合代码审查、问答等不需要修改文件的操作：

import { createReadOnlyTools } from 'pi-agent';

const tools = createReadOnlyTools(process.cwd());

// 只包含: read, grep, find, ls（4 个只读工具）
console.log(tools.map(t => t.name));  // ['read', 'grep', 'find', 'ls']
Warning

生产环境务必使用 createReadOnlyTools() 或显式配置 disabledTools。不受限的 shell_exec 是主要的安全风险。

12.5 Run Modes

InteractiveMode

交互式 REPL 模式——与 CLI 行为一致，但可编程控制：

import { InteractiveMode } from 'pi-agent';

const mode = new InteractiveMode({
    session,
    prompt: '> ',
    historyFile: '/tmp/pi-history',
});

mode.on('line', (input) => console.log('User:', input));
mode.on('response', (resp) => console.log('Agent:', resp));

await mode.start();
// 进入交互循环，按 Ctrl+C 退出

runPrintMode()

一次性运行，输出结果后退出——适合脚本和管道：

import { runPrintMode } from 'pi-agent';

await runPrintMode({
    session,
    input: 'Summarize the changes in src/',
});
// 输出到 stdout，执行完自动退出

runRpcMode()

JSON-RPC over stdio——让外部程序通过结构化协议与 Pi 通信：

import { runRpcMode } from 'pi-agent';

await runRpcMode({
    session,
    transport: 'stdio', // 或 'socket', 'pipe'
});
// 通过 stdin/stdout 收发 JSON-RPC 消息

12.6 Event Subscription

SDK 通过 typed event system 暴露运行时事件。事件类型 AgentSessionEvent 是一个 discriminated union（类型 | AgentEvent | 会话特有事件）：

import type { AgentSessionEvent, AgentSessionEventListener } from 'pi-agent';

// 订阅事件
const listener: AgentSessionEventListener = (event) => {
    switch (event.type) {
        case 'agent_start':
        case 'agent_end':
        case 'turn_start':
        case 'turn_end':
        case 'message_start':
        case 'message_update':
        case 'message_end':
        case 'tool_execution_start':
        case 'tool_execution_end':
            // ... 对应 AgentEvent 全部事件
        case 'compaction_start':
        case 'compaction_end':
        case 'session_info_changed':
        case 'thinking_level_changed':
        case 'auto_retry_start':
        case 'auto_retry_end':
            // ... 会话特有事件
    }
};

session.subscribe(listener);
// 或使用类型安全的 helper
session.on('tool_execution_start', (evt) => {
    console.error(\`[tool] \${evt.toolName}(\${JSON.stringify(evt.args)})\`);
});

所有事件类型精确对应 AgentSessionEvent union——TypeScript 会帮你做 exhaustive check。

12.7 完整示例：最简 Agent

下面是一个约 40 行的完整工作示例——用 SDK 创建一个 agent 并让它回答问题：

// minimal-agent.ts
import {
    createAgentSession,
    createReadOnlyTools,
    runPrintMode,
} from 'pi-agent';

async function main() {
    // 1. 创建会话
    const session = await createAgentSession({
        model: process.env.PI_MODEL ?? 'claude-sonnet-4-20250514',
        tools: createReadOnlyTools(process.cwd()),
        systemPrompt: 'You are a concise coding assistant.',
    });

    // 2. 订阅事件（可选）
    session.subscribe((event) => {
        if (event.type === 'tool_execution_start') {
            console.error(`[tool] ${event.toolName}(${JSON.stringify(event.args)})`);
        }
        if (event.type === 'turn_end') {
            console.error(`[turn] assistant message complete`);
        }
    });

    // 3. 运行
    await runPrintMode({
        session,
        input: process.argv[2] ?? 'What does this project do?',
    });
}

main().catch((err) => {
    console.error('Fatal:', err);
    process.exit(1);
});

运行方式：

$ npx tsx minimal-agent.ts "Explain the plugin system"

[tool] list_dir({"path": "src/plugins"})

12.8 Exercise

Exercise

Code Review Script：编写一个 Node.js 脚本，使用 Pi SDK 对指定目录下的所有 .ts 文件执行代码审查。

  1. 使用 createReadOnlyTools() 确保只读。
  2. 用 search() 找出所有 .ts 文件。
  3. 对每个文件调用 agent，传入 prompt："Review this TypeScript file for bugs, style issues, and potential improvements."
  4. 将审查结果写入 review-report.md。

提示：对每个文件，使用 runPrintMode() 传入文件内容作为 prompt。

12.9 Quiz

1. createAgentSession() 和 createAgentSessionRuntime() 的主要区别是什么？
  • 前者是同步的，后者是异步的
  • 前者是高层封装，隐藏了消息循环细节；后者暴露运行时，允许自定义消息处理
  • 前者只能用于 CLI，后者只能用于 SDK
  • 没有区别，只是名字不同
createAgentSession() 是面向大多数用户的便捷 API，内部管理了消息循环。createAgentSessionRuntime() 则把控制权交给调用者，适合需要自定义消息管道的场景。
2. 在代码审查场景中，应该使用哪个 tool factory？
  • createCodingTools()
  • createReadOnlyTools()
  • createShellTools()
  • createNetworkTools()
代码审查是只读操作。createReadOnlyTools() 只暴露读文件、列目录和搜索工具，避免 agent 意外修改文件。
3. runRpcMode() 使用什么传输方式？
  • HTTP REST
  • WebSocket
  • JSON-RPC over stdio（也支持 socket/pipe）
  • gRPC
runRpcMode() 默认使用 stdio 传输 JSON-RPC 消息，可选 socket 或 pipe。这是 IDE 集成和外部程序通信的常用方式。
4. 以下哪个事件在工具调用之前触发？
  • tool_execution_end
  • tool_execution_start
  • turn_end
  • agent_end
tool_execution_start 在 agent 决定调用工具时触发（参数已确定）。tool_execution_end 在工具执行完毕后触发。turn_end 在一整轮结束时触发。
