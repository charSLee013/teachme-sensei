建立直觉：Pi 在软件开发工作流中的位置

Main Line

先建立 Pi 的整体轮廓：它是一个终端里的编码 agent，不是单纯聊天框，也不是单个包，而是由 CLI、agent loop、provider 抽象、工具层和会话层共同组成。

Remember Now

理解 Pi 的最好入口不是背命令，而是记住它的基本循环：用户消息进入，模型决定，工具执行，结果回流，再由模型收束成回答。

1.1 开场：一个真实的 terminal 会话

让我们先看一段真实的 Pi 使用场景。你在终端中输入一行命令，Pi 就开始帮你工作了：

$ pi -p "Create a fibonacci function in fib.ts"

[Pi] Thinking...
[Pi] I'll create a fibonacci function for you.

[Tool Call] bash
  $ cat > fib.ts << 'EOF'
  /**
   * Calculate the nth Fibonacci number
   * using iterative approach.
   */
  export function fibonacci(n: number): number {
      if (n <= 0) return 0;
      if (n === 1) return 1;
      let a = 0, b = 1;
      for (let i = 2; i <= n; i++) {
          [a, b] = [b, a + b];
      }
      return b;
  }

  // Example usage
  console.log(fibonacci(10)); // 55
  EOF

[Tool Result] File created: fib.ts

[Pi] Done! I've created fib.ts with an iterative fibonacci function.
It handles edge cases (n=0, n=1) and uses O(1) space.

这段对话展示了 Pi 的核心工作循环：

  1. 你输入一个自然语言请求
  2. Pi 调用 LLM（如 Claude、GPT-4）理解你的意图
  3. LLM 决定需要执行工具调用（bash 创建文件）
  4. Pi 执行工具并返回结果
  5. LLM 确认任务完成

1.2 Pi 是什么？

Pi Agent Harness 是一个终端中的 AI 编码助手。它类似于 Claude Code、Cursor、Aider 等工具，但有几个关键区别：

Pi 的核心定位：一个开源的、自扩展的、多 LLM 提供商的交互式编码代理。它不是一个 IDE 插件，而是一个命令行工具，可以与任何编辑器配合使用。
特性 Pi 传统 Chatbot
运行环境 终端（CLI） Web 界面
工具调用 ✅ 完整工具链 ❌ 仅文本
文件操作 ✅ 读写编辑 ❌ 无
Shell 执行 ✅ 完整 bash ❌ 无
会话管理 ✅ 持久化 + 分支 ⚠️ 简单历史
LLM 选择 ✅ 35 提供商 ❌ 绑定单一
扩展系统 ✅ 插件架构 ❌ 无

1.3 核心能力一览

🤖 多 LLM 支持

35 提供商：Anthropic、OpenAI、Google、xAI、DeepSeek、Mistral、AWS Bedrock、Azure 等。通过统一 API 层调用。

🔧 工具调用

7 个内置工具：bash、edit、read、write、grep、find、ls。LLM 可以组合使用这些工具完成复杂任务。

💾 会话管理

JSONL 格式持久化，支持会话分支（fork）、树导航、多会话切换。

🧩 扩展系统

扩展系统允许你注册自定义工具、命令、键绑定、UI 组件和事件钩子。

🖥️ 三种运行模式

Interactive（TUI）、Print（单次命令）、RPC（远程调用）。

🧠 推理深度控制

6 个 thinking levels：off → xhigh。不同提供商映射到各自的推理格式。

Pi 的核心是一个 Agent Loop。这与传统 chatbot 有本质区别：

关键洞察：Agent Loop 让 AI 不再只是"回答问题"，而是"执行任务"。每次工具执行的结果都会反馈给 LLM，形成多轮交互。

1.5 快速上手

# 安装
npm install -g @earendil-works/pi-coding-agent

# 单次打印模式（最简单）
pi -p "Say exactly: ok"

# 列出可用模型
pi --list-models

# 交互模式（完整 TUI）
pi

# 指定提供商和模型
pi --provider anthropic --model claude-opus-4-7 -p "Hello"

# 查看帮助
pi --help
注意：运行 Pi 需要配置至少一个 LLM 提供商的 API Key。设置方式：环境变量（如 ANTHROPIC_API_KEY）或交互式登录。

1.6 关键术语表

Agent
一个 AI 系统，能接收目标、制定计划、调用工具、执行任务。Pi 中的 Agent 基于 LLM + 工具循环。
Tool
Agent 可以调用的函数。Pi 有 7 个内置工具 + 扩展可以注册自定义工具。每个工具有名称、描述和 JSON Schema 参数定义。
Session
一次对话的完整状态，包括消息历史、模型设置、工具调用记录。Pi 以 JSONL 格式持久化会话。
Provider
LLM 服务的提供商（如 Anthropic、OpenAI）。Pi 通过统一 API 层支持 35 提供商。
Extension
可以扩展 Pi 功能的 TypeScript 模块。可以注册工具、命令、事件钩子、UI 组件。
Compaction
当会话过长时，用 LLM 生成摘要替换旧消息，以适配上下文窗口限制。
Thinking Level
控制 LLM 的推理深度。从 off（无推理）到 xhigh（最强推理），映射到不同提供商各自的推理格式。

1.7 小测验

Q1: Pi 与传统 chatbot 的核心区别是什么？

A. 支持更多语言
B. 拥有 Agent Loop 和工具调用能力
C. 运行在 Web 浏览器中
D. 只能使用一个 LLM 提供商
✅ 正确答案：B。Pi 的核心是 Agent Loop — 它能调用工具执行真实操作，而不只是生成文本。

Q2: Pi 支持多少个 LLM 提供商？

A. 5 个
B. 12 个
C. 35 个
D. 仅 1 个
✅ 正确答案：C。Pi 通过 packages/ai/src/providers/ 支持 35 个 LLM 提供商。

Q3: 以下哪个不是 Pi 的内置工具？

A. bash
B. edit
C. deploy
D. grep
✅ 正确答案：C。Pi 的 7 个内置工具是：bash、edit、read、write、grep、find、ls。没有 deploy。

Q4: Pi 的扩展系统不能做什么？

A. 注册自定义工具
B. 订阅生命周期事件
C. 替换 Pi 的底层 LLM 调用逻辑
D. 注册斜杠命令
✅ 正确答案：C。扩展可以添加工具、命令、UI、事件钩子，但不能替换 Pi 的核心 LLM 调用逻辑（那是 pi-ai 包的事）。

1.8 练习
