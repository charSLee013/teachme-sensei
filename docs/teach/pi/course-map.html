Course Map: Pi Agent Harness Repo Course

Main line: Trace one prompt through Pi's CLI, runtime modes, session system, agent loop, provider abstraction, and tool layer without confusing repository fact with teaching simplification.

Remember now: Pi is not one monolith; it is a five-package npm workspace whose user-visible behavior emerges from `pi-coding-agent` composing `pi-agent-core`, `pi-ai`, `pi-tui`, and the experimental orchestrator boundary.

Why This Course Exists

This course exists to explain the current repository in a way a contributor can act on. The goal is not to memorize package names. The goal is to follow the control path, know where the facts live, and avoid importing a generic pnpm/Turbo or IDE-plugin mental model that this repo does not use.

Boundaries

  • This course proves current repository facts that are tied to checked files, commands, and counts.
  • This course does not prove future releases, external link uptime, or stability promises beyond the checked docs.
  • Several long code snippets are teaching reconstructions. They explain mechanisms, but they are not guaranteed to compile as-is.
  • The orchestrator chapter is architectural orientation, not a production integration manual.
